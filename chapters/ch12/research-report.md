## 二、算法设计范式系统总结

### 2.1 四大范式：适用条件与决策边界

#### 范式对比矩阵

| 范式 | 核心思想 | 适用条件 | 典型问题 | 禁忌 |
|------|----------|----------|----------|------|
| **分治** | 分解→独立求解→合并 | 子问题独立、合并简单 | 快排、归并排序、FFT | 子问题重叠（应改用DP） |
| **贪心** | 局部最优→全局最优 | 选择无后效性、最优子结构 | 最短路径、MST、活动选择 | 局部最优非全局最优 |
| **动态规划** | 子问题重叠→记忆化 | 重叠子问题、最优子结构 | LIS、背包、编辑距离 | 无重叠子问题（应改用分治） |
| **回溯/搜索** | 系统枚举→剪枝 | 无有效分解策略、需枚举 | N皇后、图着色、SAT | 剪枝不足（效率过低） |

#### 范式选择决策树

```
面对新问题：
├── 问题能否分解？
│   ├── 子问题独立？ → 分治
│   └── 子问题重叠？ → 动态规划
│
├── 问题有最优解结构？
│   ├── 局部最优→全局最优？ → 贪心（需证明）
│   ├── 有重叠子问题？ → 动态规划
│   └── 无有效分解？ → 回溯/搜索
│
├── 问题属于NP完全？
│   ├── 需精确解？ → 回溯（规模受限）
│   ├── 接受近似？ → 近似算法
│   └── 接受启发式？ → 随机化/启发式
│
└── 问题规模极大？
    ├── 数据在多台机器？ → 分布式（Ch9）
    ├── 数据持续流动？ → 流式（Ch10）
    └── 需检索融合？ → Transformer/RAG（Ch11）
```

### 2.2 分治范式：从分解到合并

#### 核心模式

```
分治三阶段：
1. Divide：将问题分解为 k 个子问题
2. Conquer：递归求解子问题（或直接求解小问题）
3. Combine：合并子问题的解

复杂度分析模板：
T(n) = k·T(n/k) + f(n)  // f(n) 是合并成本
```

#### 典型案例与前章节关联

| 案例 | 分解策略 | 合合策略 | 复杂度 | 关联章节 |
|------|----------|----------|--------|----------|
| **归并排序** | 分成两半 | 合并有序序列 | O(n log n) | Ch3 |
| **快速排序** | 分成大小两区 | 无需合并（原地） | O(n log n) 期望 | Ch3 |
| **二分查找** | 分成一半 | 无需合并（单侧） | O(log n) | Ch3 |
| **FFT** | 分奇偶项 | 合并蝶形网络 | O(n log n) | Ch3 补充 |
| **大整数乘法** | 分高低位 | Karatsuba合并 | O(n^1.585) | 扩展 |
| **矩阵乘法** | 分块 | Strassen合并 | O(n^2.807) | 扩展 |

#### 分治的边界条件

**何时不能用分治？**

```
反例1：最长递增子序列（LIS）
分解思路：求前半段LIS + 后半段LIS → 合并？
问题：后半段的LIS可能从前半段的某个元素开始
      → 子问题不独立，无法简单合并
      → 应改用动态规划

反例2：最短路径（无负边）
分解思路：求起点到中间点的最短 + 中间点到终点最短
问题：中间点选择不当 → 路径非最优
      → 子问题相互依赖
      → 但可用贪心（Dijkstra）或 DP（Bellman-Ford）

反例3：活动选择问题
分解思路：按时间分段 → 各段最优活动
问题：分段边界处活动可能跨段
      → 合并复杂
      → 应改用贪心（按结束时间排序）
```

### 2.3 贪心范式：局部最优的陷阱

#### 核心模式

```
贪心算法模板：
1. 构造选择规则（如何选出"当前最优"）
2. 证明正确性：
   - 交换论证：任何最优解都能被转换为贪心解
   - 或构造反例说明为何贪心失败
```

#### 贪心正确的两种证明模式

**模式1：交换论证（Exchange Argument）**

```
定理：贪心选择可以替换最优解中的对应选择，而不破坏最优性

证明步骤：
1. 设 OPT 是某个最优解
2. 设 G 是贪心解
3. 证明：OPT 可以被修改为 G，而仍保持最优
   （通常通过"交换"操作）
```

**模式2：贪心保持最优性（Greedy Stays Ahead）**

```
定理：贪心算法每步的选择都不比最优解差

证明步骤：
1. 设贪心解前 k 步为 G₁...Gₖ
2. 设最优解前 k 步为 O₁...Oₖ
3. 证明：G₁ ≤ O₁, G₂ ≤ O₂, ..., Gₖ ≤ Oₖ
   （贪心每步都"保持领先"）
```

#### 贪心失败的经典反例

| 问题 | 错误贪心 | 反例 | 正确解法 |
|------|----------|------|----------|
| **背包问题（0/1）** | 按性价比选 | 三物品：价值/重量=60/10, 100/20, 120/30，容量50 → 贪心选60+120=180，最优100+120=220 | DP |
| **最长路径** | 每步选最长边 | 图：A-B(5), B-C(4), A-D(3), D-C(3) → 贪心A-B-C=9，最优A-D-C=6 | 无通用解 |
| **最小生成树（无正确贪心）** | 每步选最短边连接已选集 | Kruskal/Prim是贪心但需要额外证明 | Kruskal/Prim（正确） |
| **分数背包** | 按性价比选 | 可分割 → 贪心正确 | 贪心 |

#### 与Ch7图算法的贪心对照

| 图问题 | 贪心策略 | 正确性证明 | 关联 |
|-------|---------|-----------|------|
| **Dijkstra最短路径** | 每步选距起点最近的未访问节点 | 贪心保持最优性 | Ch7 |
| **Prim最小生成树** | 每步选连接树的最小边 | 交换论证 | Ch7 |
| **Kruskal最小生成树** | 每步选最小边（不形成环） | 交换论证 | Ch7 |
| **拓扑排序（Kahn）** | 每步选入度为0的节点 | 贪心可行 | Ch7 |

### 2.4 动态规划：重叠子问题的记忆化

#### 核心模式

```
DP设计四要素：
1. 状态定义：dp[i] 或 dp[i][j] 表示什么？
2. 状态转移：dp[i] 如何由 dp[?] 计算？
3. 边界条件：dp[0] 或 dp[i][0] 等初始值
4. 计算顺序：确保依赖的子问题已求解
```

#### DP状态设计的维度选择

| 问题类型 | 状态维度 | 状态含义 | 转移来源 |
|---------|---------|---------|---------|
| **序列问题** | 一维 | dp[i] = 以i结尾的最优值 | dp[i-1], dp[i-k] |
| **区间问题** | 二维 | dp[i][j] = 区间[i,j]的最优值 | dp[i][k], dp[k+1][j] |
| **背包问题** | 二维 | dp[i][w] = 前i物品容量w的最优值 | dp[i-1][w], dp[i-1][w-w_i] |
| **网格路径** | 二维 | dp[i][j] = 到(i,j)的最优路径 | dp[i-1][j], dp[i][j-1] |
| **树形DP** | 一维（节点） | dp[u] = 以u为根的子树最优值 | dp[children] |
| **状态压缩DP** | 一维（状态） | dp[mask] = 状态mask的最优值 | dp[mask去掉某位] |

#### DP vs 分治 vs 贪心的边界

```
问题：最长公共子序列（LCS）

错误分治尝试：
分解：求前半段LCS + 后半段LCS
问题：子序列可能跨越两段 → 不独立

错误贪心尝试：
策略：每次匹配第一个相同字符
反例：
  X = "ABCD"
  Y = "ACBD"
  贪心：匹配A→无后续匹配 → LCS=1
  正确：ABD 或 ACD → LCS=3

正确DP解法：
状态：dp[i][j] = X前i个与Y前j个的LCS长度
转移：dp[i][j] = max(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]+1)
      （若X[i]=Y[j]）
```

#### DP的空间优化技巧

| 优化技术 | 适用场景 | 空间改善 | 关联章节 |
|---------|---------|---------|---------|
| **滚动数组** | 转移只依赖前一状态 | O(n) → O(1) | Ch6 |
| **状态压缩** | 01背包，转移只依赖前一物品 | O(nW) → O(W) | Ch6 |
| **单调队列优化** | 转移窗口内求最值 | O(n²) → O(n) | Ch6扩展 |
| **CDQ分治优化** | 多维DP，一维分治加速 | O(n²) → O(n log n) | 扩展 |

### 2.5 回溯与搜索：枚举的艺术

#### 核心模式

```
回溯模板：
def backtrack(state, choices):
    if is_solution(state):
        record(state)
        return
    
    for choice in choices:
        if is_valid(state, choice):  # 剪枝条件
            apply(state, choice)
            backtrack(state, next_choices)
            undo(state, choice)       # 回溯
```

#### 剪枝策略分类

| 剪枝类型 | 思想 | 典型应用 |
|---------|------|----------|
| **可行性剪枝** | 当前状态无法到达解 | N皇后：同列已有皇后 |
| **最优性剪枝** | 当前解不可能优于已知最优 | 分支限界法 |
| **对称性剪枝** | 等价状态只搜索一次 | 图着色：颜色交换对称 |
| **记忆化剪枝** | 已搜索状态跳过 | 搜索+哈希表 |

#### 回溯 vs DP 的选择边界

| 问题特征 | 回溯适用 | DP适用 |
|---------|---------|---------|
| **解空间结构** | 无重叠子问题 | 有重叠子问题 |
| **解的形式** | 所有可行解 | 最优解 |
| **规模** | 通常较小（剪枝后） | 可较大（状态压缩） |
| **典型问题** | N皇后、图着色、SAT | LIS、背包、编辑距离 |

## 三、问题建模技巧

### 3.1 建模的核心思维

#### 建模的本质

> **建模 = 将现实问题翻译为算法问题的语言**

```
现实问题 → 算法问题：
├── 输入：现实世界的"数据" → 算法世界的"结构"
├── 输出：现实世界的"答案" → 算法世界的"目标"
├── 约束：现实世界的"限制" → 算法世界的"条件"
└── 目标：现实世界的"好坏" → 算法世界的"优化方向"
```

#### 建模的四步骤

```
Step 1: 抽象化
- 识别关键对象（人、物品、地点、时间）
- 确定对象之间的关系

Step 2: 结构化
- 选择数据结构表示对象和关系
- 例：数组、链表、树、图、矩阵

Step 3: 算法化
- 将现实操作翻译为算法操作
- 例：匹配、排序、遍历、搜索、优化

Step 4: 验证化
- 用小例子验证建模是否正确
- 检查边界情况
```

### 3.2 常见建模模式

#### 模式1：排序建模

```
现实问题："安排会议顺序使得等待时间最小"
建模：
├── 输入：会议列表（开始时间、持续时间）
├── 输出：会议顺序
├── 约束：无冲突
├── 目标：最小化等待时间总和
└── 算法：按开始时间或持续时间排序（贪心）

典型应用：
├── 任务调度
├── 活动选择（Ch5贪心）
├── 排课问题
└── 排队优化
```

#### 模式2：查找建模

```
现实问题："在百万商品中找到用户想要的"
建模：
├── 输入：商品列表（属性、关键词）
├── 输出：匹配的商品
├── 约束：时间限制
├── 目标：精确或近似匹配
└── 算法：哈希表、B树、向量检索（Ch3 + Ch11）

典型应用：
├── 搜索引擎
├── 推荐系统
├── 数据库查询
└── RAG检索（Ch11）
```

#### 模式3：图建模

```
现实问题："找出地铁换乘最少路线"
建模：
├── 输入：地铁站列表、换乘关系
├── 输出：换乘路线
├── 约束：时间限制、票价限制
├── 目标：换乘最少或时间最短
└── 算法：BFS（换乘最少）或Dijkstra（时间最短）

典型应用：
├── 社交网络（Ch7）
├── 导航系统（Ch7最短路径）
├── 依赖关系（Ch7拓扑排序）
├── 知识图谱（Ch7图结构）
└── 注意力矩阵（Ch11 Transformer）
```

#### 模式4：流建模

```
现实问题："分配任务给员工，最大化完成量"
建模：
├── 输入：任务列表、员工列表、任务-员工匹配度
├── 输出：任务分配方案
├── 约束：员工能力限制、任务需求
├── 目标：最大化完成量或满意度
└── 算法：最大流（Ch7扩展）

典型应用：
├── 任务分配
├── 资源调度
├── 二分图匹配
└── 网络流量控制
```

### 3.3 建模陷阱与注意事项

#### 陷阱1：过度抽象

```
问题：设计校园导航系统

过度抽象建模：
├── 将所有地点视为同质节点
├── 所有路径视为同质边
└── 问题：忽略了楼梯、坡道、时段限制

正确建模：
├── 区分步行路径和车辆路径
├── 标注路径属性（长度、类型、时段）
└── 针对不同用户类型设计不同算法
```

#### 陷阱2：忽略约束

```
问题：排课系统

错误建模：
├── 只考虑时间冲突
└── 忽略：教室容量、教师偏好、先修课程

正确建模：
├── 多约束建模（时间、空间、依赖）
├── 可能需要多目标优化
└── 或简化问题（固定部分约束）
```

#### 陷阱3：建模粒度不当

```
问题：物流路径规划

粒度过粗：
├── 只考虑城市级别节点
└── 忽略：街区、时段、载重

粒度过细：
├── 每条街道每个时段建模
└── 问题：规模爆炸，求解困难

正确粒度：
├── 根据求解目标和数据规模选择粒度
├── 阶段性建模（城市→街区→街道）
└── 或分层建模（Ch9分布式分片）
```

## 四、复杂度分析方法

### 4.1 时间复杂度层次

#### 复杂度等级与规模关系

| 复杂度 | n=10 | n=100 | n=1000 | n=10⁶ | 实际意义 |
|--------|------|-------|--------|-------|----------|
| **O(1)** | ✓ | ✓ | ✓ | ✓ | 常数操作 |
| **O(log n)** | ✓ | ✓ | ✓ | ✓ | 二分查找 |
| **O(n)** | ✓ | ✓ | ✓ | ✓ | 线性扫描 |
| **O(n log n)** | ✓ | ✓ | ✓ | ✓秒级 | 高效排序 |
| **O(n²)** | ✓ | ✓秒级 | ✓分钟 | ✓小时 | 两层循环 |
| **O(n³)** | ✓秒级 | ✓分钟 | ✓小时 | ✗ | 矩阵运算 |
| **O(2ⁿ)** | ✓秒级 | ✗ | ✗ | ✗ | 组合枚举 |
| **O(n!)** | ✓秒级 | ✗ | ✗ | ✗ | 全排列 |

#### 复杂度分析的三大技术

| 技术 | 适用场景 | 方法 |
|------|----------|------|
| **基本计数** | 直接循环 | 计算循环次数 |
| **递归分析** | 分治、递归 | 递推方程 + 主定理 |
| **摊还分析** | 数据结构操作 | 平均化代价 |

### 4.2 主定理（Master Theorem）

#### 形式

```
T(n) = aT(n/b) + f(n)

其中：
a ≥ 1：子问题个数
b > 1：子问题规模缩小比例
f(n)：合并成本

三种情况：
Case 1: f(n) = O(n^(log_b a - ε)) → T(n) = Θ(n^log_b a)
        （合并成本小于子问题成本）

Case 2: f(n) = Θ(n^log_b a) → T(n) = Θ(n^log_b a · log n)
        （合并成本与子问题成本相当）

Case 3: f(n) = Ω(n^(log_b a + ε)) → T(n) = Θ(f(n))
        （合并成本大于子问题成本）
```

#### 应用案例

| 算法 | 递推方程 | a/b | f(n) | 主定理情况 | 结果 |
|------|----------|------|------|-----------|------|
| **归并排序** | T(n) = 2T(n/2) + n | a=2, b=2 | n | Case 2 | Θ(n log n) |
| **二分查找** | T(n) = T(n/2) + 1 | a=1, b=2 | 1 | Case 2 | Θ(log n) |
| **快排（平均）** | T(n) = 2T(n/2) + n | a=2, b=2 | n | Case 2 | Θ(n log n) |
| **Karatsuba乘法** | T(n) = 3T(n/2) + n² | a=3, b=2 | n² | Case 3 | Θ(n²) |

### 4.3 摊还分析（Ch2 关联）

#### 三种方法对比

| 方法 | 思想 | 适用场景 | 典型应用 |
|------|------|----------|----------|
| **聚合分析** | 计算n次操作总代价÷n | 操作序列有规律 | 动态数组扩容 |
| **核算分析** | 每次操作预付"信用" | 代价波动大 | 栈的multipop |
| **势能分析** | 定义势能函数Φ(D) | 状态变化规律 | 并查集路径压缩 |

#### 动态数组扩容的摊还分析

```
操作：动态数组append，容量不足时扩容2倍

单次操作代价：
├── 正常：O(1)
├── 扩容：O(n)（复制所有元素）

摊还分析（聚合）：
├── 假设从空数组开始，插入n个元素
├── 扩容发生在：1, 2, 4, 8, ..., n/2
├── 总扩容代价：1+2+4+...+n/2 < n
├── 加上n次插入代价：O(1)×n
├── 总代价：O(n) + O(n) = O(n)
└── 摊还代价：O(n)/n = O(1)

结论：虽然单次扩容是O(n)，但摊还代价是O(1)
```

## 五、算法正确性证明

### 5.1 证明方法分类

| 方法 | 适用算法 | 核心思想 | 典型应用 |
|------|----------|----------|----------|
| **循环不变式** | 循环算法 | 建立迭代过程中的恒成立性质 | 排序、查找 |
| **归纳证明** | 递归/DP | 证明小规模成立，推广到大规模 | 分治、DP |
| **交换论证** | 贪心 | 证明贪心选择可替换最优解的对应选择 | MST、最短路径 |
| **反例构造** | 算法错误证明 | 构造输入使算法输出错误 | 贪心失败案例 |
| **边界验证** | 所有算法 | 检查边界情况的正确性 | 空输入、单元素 |

### 5.2 循环不变式

#### 模板

```
循环不变式定义：
在循环的每次迭代开始/结束时，某个性质P始终成立。

证明步骤：
1. 初始化：循环开始前，P成立
2. 保持：若某次迭代开始时P成立，迭代结束后P仍成立
3. 终止：循环结束时，P + 终止条件 → 算法正确
```

#### 案例：插入排序

```
插入排序伪代码：
for i = 1 to n-1:
    key = A[i]
    j = i-1
    while j >= 0 and A[j] > key:
        A[j+1] = A[j]
        j--
    A[j+1] = key

循环不变式：
P: A[0..i-1] 在每次外循环迭代结束时是已排序的

证明：
├── 初始化：i=1，A[0..0]只有1个元素，已排序 ✓
├── 保持：插入A[i]到正确位置，A[0..i]已排序 ✓
└── 终止：i=n，A[0..n-1]已排序 → 整个数组已排序 ✓
```

### 5.3 归纳证明

#### 模板

```
数学归纳法：
1. 基础情况：证明n=0或n=1时成立
2. 归纳假设：假设n=k时成立
3. 归纳步骤：证明n=k+1时成立

算法归纳证明：
将算法的正确性归纳到问题规模
```

#### 案例：快速排序（分区正确性）

```
Claim: Partition将数组分为三部分：<pivot, =pivot, >pivot

归纳证明：
├── 基础：数组长度为1，pivot就是该元素，分区正确
├── 归纳假设：假设对长度≤k的数组分区正确
├── 归纳步骤：
│   ├── 设数组长度k+1
│   ├── Partition扫描：i从左到右，j从右到左
│   ├── 每次交换保证：A[left..i] ≤ pivot, A[j..right] ≥ pivot
│   └── 最终i和j相遇，分区完成 ✓
└── 结论：所有规模下分区正确
```

### 5.4 交换论证（贪心证明）

#### 案例：活动选择问题

```
Claim: 按结束时间最早选择活动，是最优解

交换论证：
├── 设贪心解G选了活动{g₁, g₂, ..., gₖ}，按结束时间排序
├── 设最优解OPT选了活动{o₁, o₂, ..., oₘ}, m ≥ k
├── 证明：OPT可以被转换为G
│   ├── Step 1: g₁是结束最早的，所以g₁ ≤ o₁（结束时间）
│   ├── Step 2: 将OPT的o₁替换为g₁
│   │   → OPT仍可行（因为g₁结束更早，不冲突）
│   ├── Step 3: 重复上述替换，每次都将OPT的当前活动替换为G的对应活动
│   └── 最终：OPT = G，且G也是最优解 ✓
└── 结论：贪心算法正确
```

## 六、六问诊断法系统回顾

### 6.1 六问详解

#### Q1：我真的理解问题吗？

```
检查清单：
├── 输入是什么？（类型、规模、格式）
├── 输出是什么？（形式、要求）
├── 约束是什么？（时间、空间、正确性）
├── 有小例子吗？（手动计算验证）
└── 有等价表述吗？（重述问题）

常见错误：
├── 混淆输入输出
├── 忽略隐式约束
├── 未构造小例子
```

#### Q2：有没有简单正确的算法？

```
检查清单：
├── 暴力解法是什么？（枚举所有可能性）
├── 暴力解法的复杂度？
├── 暴力解法慢在哪里？
└── 有启发式基线吗？

意义：
├── 验证问题理解是否正确
├── 作为正确性对照
├── 发现瓶颈所在
```

#### Q3：我的问题在已知目录中吗？

```
已知问题目录：
├── 排序问题（Ch3）
├── 查找问题（Ch3）
├── 图问题（Ch7）
│   ├── 遍历（BFS/DFS）
│   ├── 最短路径
│   ├── MST
│   ├── 拓扑排序
│   └── 流问题
├── 组合搜索（Ch5回溯）
├── 优化问题（Ch6 DP, Ch7贪心）
├── NP完全问题（Ch8）
├── 分布式问题（Ch9）
├── 流式问题（Ch10）
└── 检索融合问题（Ch11）

检查方法：
├── 能否映射到已知问题？
├── 是否是已知问题的变体？
└── 是否组合多个已知问题？
```

#### Q4：特殊情况是否可解？

```
检查清单：
├── 固定一个参数，问题是否变简单？
├── 缩小规模，能否看清结构？
├── 删掉一个约束，能否求解？
├── 特殊输入（有序、空、单元素）？

意义：
├── 发现问题的隐藏结构
├── 获得启发和洞察
└── 为推广做准备
```

#### Q5：哪种范式最像？

```
范式匹配检查：
├── 能分解？ → 分治（子问题独立）
├── 能贪心？ → 需证明无后效性
├── 有重叠子问题？ → DP
├── 无有效分解？ → 回溯/搜索
├── NP完全？ → 近似/启发式
├── 规模极大？ → 分布式/流式

对比方法：
├── 每种范式给出一个候选方案
├── 分析各方案的复杂度
└── 选择最优适配
```

#### Q6：还是想不出怎么办？

```
策略：
├── 回到问题定义，重新审视
├── 构造反例，检验假设
├── 查阅文献/教程
├── 请教专家或同伴
└── 让Agent生成候选方案，逐一审查

Agent协作模式：
├── 输入问题描述
├── Agent走六问流程
├── 输出结构化诊断报告
├── 人工审查每步判断
└── 修正并迭代
```
