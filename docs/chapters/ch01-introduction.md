# 第1章 问题建模与算法思维

## 1.1 算法是一种技术

算法与硬件同等重要。即使慢 1000 倍的计算机用归并排序，也能在 4 小时内完成快计算机用插入排序需要 23 天的工作量。算法不是代码，是解决问题的思维方式。

CLRS 的核心论点：选择高效算法的收益可以远超硬件升级。

## 1.2 循环不变式——证明算法正确的系统方法

初始化、保持、终止三步证明模板。这不是灵感驱动的——任何循环都能用循环不变式证明正确性。与反例证伪互补：反例证伪，循环不变式证真。

**人的能力 vs AI 的能力**：人擅长判断循环不变式是否成立（初始化对吗？保持对吗？终止时能得到什么？），AI 擅长枚举所有路径验证。循环不变式是评判 LLM 代码输出的形式工具——要求 AI 声明不变式，人检查三步是否成立。

## 1.3 反例构造——证明算法错误的核心能力

Skiena 的教学核心： Robot Tour 问题展示错误算法的荒谬性。证明算法错误的最好方法是构造反例——这是评判 LLM 输出的核心能力。

反例构造与循环不变式形成互补：反例找错，不变式证对。两者都是人必须掌握的，因为 LLM 经常生成"看起来对但有微妙错误"的代码。

## 1.4 渐近分析——从增长量级到下界

Θ/O/Ω/o/ω 是函数集合而非记号。三值比较律不成立（n 与 n^{1+sin n} 不可比）。增长量级分类：O(1)到O(2^N)，从 N² 到 N log N 不是微调而是质变。

**指示器随机变量**：E[I{A}] = Pr{A}，线性性即使不独立也成立。贯穿6章的分析利器——雇佣问题、快排期望、桶排序都依赖它。

## 1.5 建模——从现实到算法

Skiena 的"设计图而非算法"原则。_combinatorial objects_（排列、子集、图、树）和递归对象。建模失误比算法失误代价更高。

## 1.6 算法设计的自我质询

Skiena 六问诊断法：(1)我真的理解问题吗？(2)简单算法或启发式？(3)查目录？(4)特殊情况可解？(5)哪种范式？(6)还是想不出→循环。

试飞员比喻：即将坠毁也不恐慌，跑过可能行动的清单。正确回答"能不能"不是"不能"而是"不能，因为……"。

**这六问本身就是可编码的算法——设计算法的算法。** 终章将回到这里，把这六问编码为 Agent skill。

## 1.7 练习

**层次一（基础）**：
1. 用循环不变式证明插入排序的正确性
2. 构造反例证明贪心硬币找零策略不可行
3. 给出 O(n) 和 O(n²) 之间所有常见增长量级，并排序

**层次二（LLM 协同）**：
4. 让 LLM 实现一个排序算法，用循环不变式检查其 PARTITION 过程是否正确
5. 用六问法分析一个你不熟悉的问题（如活动选择），观察哪些步骤 LLM 能回答、哪些不能

**层次三（Agent 设计）**：
6. 设计一个 Skill：输入问题描述，执行六问法的第3步（查问题目录），输出问题分类和推荐范式
