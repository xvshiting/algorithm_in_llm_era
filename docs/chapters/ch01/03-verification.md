# 1.3 如何验证正确性

当你让 coding agent 写了一段代码，你怎么判断它对不对？

很多人会用几个测试用例跑一下，通过了就觉得没问题。但测试用例只能证明"有错"，不能证明"没错"。你的测试用例可能漏掉了关键的边界情况。

你需要更系统的方法来验证正确性。这一节我们学习两种武器：循环不变式用来证明正确，反例用来证明错误。

---

## 循环不变式：证明算法正确的系统方法

循环不变式是一个在循环的每次迭代前后都保持为真的命题。用它证明算法正确需要三步。

第一步，初始化：证明循环开始前，不变式成立。

第二步，保持：假设某次迭代前不变式成立，证明迭代后仍成立。

第三步，终止：证明循环结束时，不变式给出期望的结果。

这三步缺一不可。如果初始化或保持有问题，不变式可能从未成立过。如果终止条件不对，不变式成立但得不到期望结果。

---

## 一个例子：插入排序

让我们用循环不变式证明插入排序的正确性。

插入排序的思路很简单：从第二个元素开始，把每个元素插入到前面已排序部分的正确位置。

它的循环不变式是：在外层循环的每次迭代开始时，数组的 A[1..j-1] 部分包含原来在 A[1..j-1] 中的元素，且已经排好序。

初始化：当 j=2 时，A[1..1] 只有一个元素，自然是有序的。不变式成立。

保持：假设某次迭代开始时 A[1..j-1] 有序。内层循环把 A[j] 插入到正确位置，插入后 A[1..j] 有序。下一次迭代开始时，j 变成 j+1，不变式变成 A[1..j] 有序，这正是我们刚刚保证的。

终止：循环结束时 j=n+1。根据不变式，A[1..n] 有序。这正是我们想要的。

证明完成。

---

## 为什么循环不变式在 agent 时代更重要

因为 agent 给你的代码"看起来对"。

它可能通过大部分测试用例。但边界情况呢？空数组？单元素？已经排序？逆序？所有元素相同？这些情况 agent 可能没想过。

如果你让 agent 在写代码的同时写出循环不变式，你就有了检验它正确性的工具。如果它写不出循环不变式，说明它自己也不确定代码对不对。如果循环不变式有问题，那代码肯定有问题。

你可以这样问 agent："请写出这个算法的循环不变式，并证明初始化、保持、终止三步。"

如果 agent 能清晰回答，代码可信度就高很多。如果支支吾吾，你就得小心了。

---

## 反例：证明算法错误的最快方法

有时候你想知道的不是"怎么证明正确"，而是"这个算法对不对"。

如果一个算法是错的，最快的方法是找一个反例：一个让算法失败的具体输入。

构造反例需要你理解算法的逻辑。你要问：这个算法在什么情况下会做出错误的决定？

贪心算法特别容易出错。比如"每次选价值最高的物品"这个策略，在背包问题上就不对。假设背包容量是 10，有三个物品：A 价值 60 重量 6，B 价值 50 重量 5，C 价值 50 重量 5。贪心策略会选 A（价值最高），然后背包只剩容量 4，装不下 B 或 C。总价值 60。但最优解是选 B 和 C，总价值 100。

这个反例告诉我们：贪心策略在这个问题上不对。原因是什么？贪心只看局部，不看全局。选了 A 之后，浪费了很多空间。

---

## 反例构造的技巧

构造反例有一些常见模式。

边界情况：空输入、单元素、最小值、最大值。很多算法在边界情况没有处理好。

特殊结构：已经排序、完全逆序、所有元素相同。这些特殊结构可能触发算法的退化。

对称性破坏：贪心策略经常在需要"回溯"的地方失败。比如需要先选一个看起来不优的选择，才能达到更优的全局结果。

当你怀疑一个算法有问题时，试着构造这些情况。往往能发现问题。

---

## 实践：检验一个 agent 生成的排序实现

假设你让 agent 实现排序，它给了你这段代码：

```
def sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(i + 1, n):
            if arr[i] > arr[j]:
                arr[i], arr[j] = arr[j], arr[i]
    return arr
```

第一步，识别算法。这是选择排序。每次外层循环选出最小元素放到当前位置。

第二步，检查边界情况。空数组返回空数组，正确。单元素数组返回自己，正确。已经排序的数组不变，正确。

第三步，分析复杂度。外层循环 n 次，内层循环平均 n/2 次，总比较次数约 n²/2。复杂度 O(n²)。

第四步，判断是否满足需求。如果数据规模是几千，O(n²) 可能还可以接受。如果是几万以上，可能太慢了。

第五步，考虑是否需要更快的算法。如果确实太慢，你需要告诉 agent："给我一个 O(n log n) 的实现"，然后检验新代码。

---

## 小结

验证正确性有两件武器：循环不变式证明对，反例证明错。

在 agent 时代，这两件武器更重要。因为 agent 会给你"看起来对"的代码，你需要有能力判断它是不是真的对。

让 agent 写出循环不变式，是一个很好的检验方法。如果它能清晰写出，说明它理解自己写的代码。如果写不出，你就要小心了。
