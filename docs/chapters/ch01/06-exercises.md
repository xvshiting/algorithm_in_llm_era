# 1.6 练习

这一章的练习不是让你"实现一个算法"，而是让你"做出判断"。这才是 agent 时代需要的能力。

---

## 基础练习

**练习 1.1**

用循环不变式证明：在一个已排序数组中插入一个元素后，数组仍然有序。

要求写出：循环不变式是什么，以及初始化、保持、终止三步的证明。

提示：这个问题比插入排序更简单。你只需要证明插入操作保持有序性，不需要处理整个排序过程。

**练习 1.2**

构造反例证明："先选眼前看起来最好的选择"不一定得到整体最好的结果。

场景：你有 10 元预算，必须从三件商品中选两件，目标是在不超过预算的前提下让总价尽可能高。

商品价格如下：A 价格 8，B 价格 6，C 价格 4。

请比较两种做法：

1. 先选最贵的商品 A，再尝试选第二件。
2. 选择 B 和 C。

说明哪个结果更好，并解释为什么"先选眼前最贵"这个规则会失败。

**练习 1.3**

分析以下代码的最坏时间复杂度：

```
def mystery(arr):
    n = len(arr)
    result = []
    for i in range(n):
        inserted = False
        for j in range(len(result)):
            if arr[i] < result[j]:
                result.insert(j, arr[i])
                inserted = True
                break
        if not inserted:
            result.append(arr[i])
    return result
```

提示：注意 `list.insert` 操作本身的复杂度，以及内外循环的执行次数。

## 进阶练习

**练习 1.4**

找一个你熟悉的问题（比如"安排会议时间避免冲突"），用六问法完整分析一遍。

然后让 coding agent 帮你实现。

你的描述足够清晰吗？agent 理解的是你想要的问题吗？如果有偏差，问题出在哪里？

记录你的问题描述，agent 的理解，以及实际得到的代码。分析哪里可以改进。

**练习 1.5**

让 coding agent 实现一个有序数组查找函数。要求它利用"数组已经有序"这个条件，每次检查中间位置，并排除不可能包含目标的一半范围。

然后用以下测试用例验证：

空数组，目标任意。

单元素数组，目标等于元素。

单元素数组，目标不等于元素。

目标是数组的第一个元素。

目标是数组的最后一个元素。

目标不存在于数组中。

数组有重复元素，目标是重复元素之一。

记录 agent 犯的错误（如果有）。用循环不变式分析为什么错误会发生。

**练习 1.6**

给定约束：数据规模最多 10⁶ 个元素，需要在 1 秒内完成。

判断以下复杂度是否可接受：O(n)，O(n log n)，O(n√n)，O(n²)，O(2ⁿ) 假设 n ≤ 30。

解释你的判断依据。

## 挑战练习

**练习 1.7**

描述一个你日常工作或生活中的问题，将其建模为某种算法问题。

要求：

明确输入、输出、约束。

识别问题结构（搜索/优化/判定/构造）。

归类到已知问题结构（排序、查找、对象关系、选择方案等）。如果你不确定，就写出你不确定的原因。

让 coding agent 根据你的描述实现代码。

评估：你的描述足够清晰吗？agent 的实现对吗？如果不对，是建模错了还是描述不够清楚？

---

## 小结

这一章的练习重点是判断能力。

判断问题是什么。判断算法对不对。判断复杂度够不够。判断建模对不对。

这些能力在 agent 时代更重要。因为 agent 可以帮你实现，但不能帮你判断。

下一章，我们会学习数据结构。但重点仍然不是"怎么实现"，而是"怎么选择"、"怎么判断"。
