# 第2章 数据结构——抽象接口与扩张

## 2.1 数据结构是算法的副产品

Sedgewick 的核心观点：数据结构不是独立存在的，而是算法设计过程中自然产生的工具。栈支持 DFS，队列支持 BFS，优先队列支持 Dijkstra——数据结构是算法的"需求规格"。

CLRS 的补充：**数据结构扩张方法论**（定理14.1）——不是发明新结构，而是系统性增强已有结构。如果属性 f 在节点 x 上仅依赖 x 及其子节点（及其 f 值），则 f 可在 O(lg n) 内维护。

## 2.2 栈与队列

LIFO/FIFO 抽象。栈的应用：括号匹配、表达式求值、DFS。队列的应用：BFS。

## 2.3 链表与指针

单链表、双链表、循环链表。哨兵节点简化边界条件。自由列表对象管理（ALLOCATE-OBJECT/FREE-OBJECT）是微缩内存分配器。

## 2.4 散列表——从数据结构到随机化算法

**链式散列**：每个槽维护链表，Θ(1+α) 查找期望。**开放寻址**：线性探测（简单但聚集）、二次探测（消除初次聚集）、双散列（两步哈希消除二次聚集）。删除用墓碑标记而非物理删除。装载因子 α > 0.7 时性能急剧下降。

Skiena 视角：哈希是**随机化算法**——balls-and-bins 分析、Bloom 过滤器、MinHash。CLRS 视角：**全域散列的形式化证明**——H_{pm} 族构造，碰撞概率 ≤ 1/m。

三个视角的交汇：实现、概率直觉、数学证明。Bloom 过滤器在 RAG 管道中用于去重和缓存过滤——"无假阴性"意味着永远不遗漏相关文档。

## 2.5 二叉搜索树与平衡树

BST 查找/插入/删除 O(h)。最坏 O(n)，随机 BST 期望高度 O(lg n)（CLRS 用指数高度 Y_n = 2^{X_n} 和 Jensen 不等式证明）。红黑树保证 O(lg n)——五条性质（根黑、叶黑、红子黑、黑高相同、所有路径黑节点数相同）。插入三种情况：红叔→重着色、黑叔（LL/RR）→单旋转、黑叔（LR/RL）→双旋转。删除的"双重黑色"概念处理复杂情况——CLRS 独有的教学装置。

## 2.6 优先队列与堆

堆排序的精妙点：BUILD-MAX-HEAP 的线性时间证明（高度分布 ⌈n/2^{h+1}⌉，总代价 O(n)）。优先队列支持 Dijkstra 和事件驱动模拟。

## 2.7 并查集——几乎常数时间的集合合并

按秩合并 + 路径压缩 → O(mα(n))，α(n) ≤ 4 对所有实际 n。这是宇宙中最慢增长的非恒定函数——看似简单的数据结构需要 Ackermann 函数刻画其复杂度。并查集支持 Kruskal 的 MST 算法（第4章引用）、等价类划分、连通分量维护。

## 2.8 摊还分析——超越最坏情况的分析维度

三书中唯有 CLRS 覆盖。三种方法：聚合分析、核算法、势能法。动态表收缩的抖动问题——朴素策略（α < 1/2 时减半）导致 Θ(n²)，正确策略是 α < 1/4 时减半。势函数 Φ = {2num-size (α≥1/2), size/2-num (α<1/2)} 使所有操作 O(1) 摊还。

**关键区别**：摊还分析给出的是**最坏情况保证**，不是期望值——不依赖概率假设。

## 2.9 练习

**层次一（基础）**：
1. 证明 BUILD-MAX-HEAP 的线性时间复杂度
2. 给出红黑树插入的三种情况，画出旋转和重着色过程
3. 用势能法分析动态表扩张的摊还代价

**层次二（LLM 协同）**：
4. 让 LLM 实现建堆操作，验证其 MAX-HEAPIFY 的循环不变式
5. 给定一个应用场景（搜索引擎倒排索引、任务调度、LRU 缓存），让 LLM 推荐数据结构并分析摊还复杂度

**层次三（Agent 设计）**：
6. 设计一个 Skill：根据应用场景（操作类型、规模、延迟要求）自动推荐数据结构并生成实现框架——这正是数据结构扩张方法论的应用
