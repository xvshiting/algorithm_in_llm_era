## War Story：循环不变式拯救了代码审查

2024年，某团队要实现一个自定义排序。工程师小王让 Claude 写了一个插入排序的变体：

```python
def insertion_sort_variant(arr):
    for i in range(len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

小王觉得代码有问题——`for i in range(len(arr))` 应该从 1 开始，而不是 0。

他是对的。当 i = 0 时，j = -1，while 循环不执行，`arr[-1 + 1] = arr[0]`，看起来没问题……但仔细想，第一个元素不需要"插入"。

小王用循环不变式验证了这个问题。循环不变式不仅帮助理解代码，还帮助发现 bug。

---

## 插入排序的基本思想

插入排序模仿整理扑克牌：拿起一张牌，插入到已整理好的牌堆中的正确位置。

**算法步骤**：

```
for i = 1 to n-1:
    key = A[i]
    j = i - 1
    while j >= 0 and A[j] > key:
        A[j + 1] = A[j]  // 向右移动
        j -= 1
    A[j + 1] = key  // 插入到正确位置
```

**图解说明**：

```
初始：[5, 2, 4, 6, 1, 3]

i=1: [2, |5, 4, 6, 1, 3]  ← 把 2 插入到 [5] 前面
i=2: [2, 4, |5, 6, 1, 3]  ← 把 4 插入到 [2, 5] 中间
i=3: [2, 4, 5, |6, 1, 3]  ← 把 6 插入到 [2, 4, 5] 后面
i=4: [1, 2, 4, 5, |6, 3]  ← 把 1 插入到最前面
i=5: [1, 2, 3, 4, 5, |6]  ← 把 3 插入到中间

结果：[1, 2, 3, 4, 5, 6]
```

竖线 `|` 左边是已排序部分，右边是未排序部分。

## 循环不变式证明

循环不变式是证明算法正确性的核心方法。它有三步：

1. **初始化**：循环第一次迭代前，不变式成立
2. **保持**：每次迭代后，不变式仍然成立
3. **终止**：循环结束时，不变式给出我们想要的结论

**插入排序的循环不变式**：

> 在每次 for 循环迭代开始时，子数组 A[0..i-1] 包含原来在 A[0..i-1] 的元素，且已排序。

### 初始化

i = 1 时，子数组 A[0..0] 只有一个元素，必然已排序。不变式成立。

### 保持

假设迭代开始时，A[0..i-1] 已排序。

算法把 A[i] 插入到 A[0..i-1] 的正确位置：
- 找到位置 j，使得 A[0..j] ≤ key < A[j+1..i-1]
- 把 A[j+1..i-1] 向右移动一位
- 把 key 放到位置 j+1

迭代结束后，A[0..i] 已排序。不变式成立。

### 终止

循环结束条件：i = n。此时 A[0..n-1] 已排序。这就是整个数组已排序——正确性证明完成。

## 复杂度分析

**时间复杂度**：

| 情况 | 比较次数 | 移动次数 | 总复杂度 |
|------|----------|----------|----------|
| **最好（已排序）** | n-1 | 0 | Θ(n) |
| **最坏（逆序）** | Σ_{i=1}^{n-1} i = n(n-1)/2 | Σ_{i=1}^{n-1} (i+1) | Θ(n²) |
| **平均（随机）** | Θ(n²/4) | Θ(n²/4) | Θ(n²) |

**为什么平均是 Θ(n²/4)**？

引入**倒置数**的概念：倒置是数组中逆序的一对元素 (i, j)，其中 i < j 但 A[i] > A[j]。

- 最坏情况：n 个元素有 n(n-1)/2 个倒置
- 最好情况：0 个倒置
- 随机排列：期望倒置数 = n(n-1)/4

**为什么？** n(n-1)/2 对元素，每对有 1/2 概率是倒置。

插入排序每次消除一个倒置，需要 Θ(1) 操作。期望倒置数 Θ(n²/4)，所以期望复杂度 Θ(n²)。

## 插入排序的适用场景

插入排序不是最快的，但它在特定场景很有用：

| 场景 | 为什么适合插入排序 |
|------|-------------------|
| **小规模数据** | n < 50 时，Θ(n²) 和 Θ(n lg n) 差别不大 |
| **几乎有序的数据** | 倒置数少时，接近 Θ(n) |
| **在线排序** | 数据逐步到达时，可以增量插入 |
| **作为其他排序的子过程** | 快排、归并排序在小数组时切换到插入排序 |

## 与 LLM 的协作要点

当让 agent 实现插入排序时，你要检查：

1. **循环不变式是否正确**：
   - 内循环条件是 `j >= 0` 还是 `j > 0`？
   - `arr[j + 1] = key` 是否在正确位置？

2. **边界条件**：
   - 空数组？单元素数组？
   - 全相等元素？

3. **复杂度判断**：
   - agent 是否知道这是 Θ(n²) 算法？
   - 是否建议在小数据时使用？

**审查技巧**：让 agent 先写代码，然后用循环不变式形式化验证。发现差异就是 bug。

## 练习

### 层次一（基础）

1. **循环不变式验证**：证明插入排序内循环的循环不变式：
   > while 循环开始时，A[j+1..i] 的元素都大于 key，且 A[0..j] 已排序。

2. **倒置数计算**：计算数组 [3, 1, 4, 1, 5, 9, 2, 6] 的倒置数。

3. **复杂度验证**：对 n = 10 的逆序数组，计算插入排序需要多少次比较和移动。

### 层次二（LLM 协同）

4. **代码审查**：让 LLM 写一个插入排序的实现，用循环不变式验证其正确性，找出可能的 bug。

5. **变体分析**：让 LLM 实现二分插入排序（用二分查找找插入位置），分析其时间复杂度。

### 层次三（Agent 设计）

6. **在线排序 Skill**：设计一个 Skill，处理逐步到达的数据流，使用插入排序维护一个有序列表。
