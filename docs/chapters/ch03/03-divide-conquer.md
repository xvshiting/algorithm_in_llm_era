## 学习目标

学完这一节，你能：

- 理解分治策略的设计范式
- 分析归并排序和快速排序的复杂度
- 使用循环不变式证明算法正确性
- 理解为什么"确定性"有时是敌人

---

## 开场：一个被"攻击"的排序算法

同学们，我先讲一个有趣的故事。

1999年，有人发现一个致命问题：某些快排实现可以被"杀手序列"攻击。

构造一个特殊输入，使得 qsort() 从 Θ(n lg n) 退化到 Θ(n²)。攻击原理很简单：利用确定性快排的 pivot 选择策略（比如"选第一个元素"），构造一个总是选到最小或最大的输入。

这告诉我们什么？

**确定性快排依赖于输入分布**。要消除这种依赖，必须引入随机性。

这不是"bug"，是设计的本质。理解这一点，你就会明白为什么随机化是一种设计哲学，而不仅仅是"运气不好时的补救"。

---

## 分治范式：分解、解决、合并

分治策略听起来很抽象，但其实就是一个简单的三步骤：

1. **分解（Divide）**：把问题分成若干子问题
2. **解决（Conquer）**：递归解决子问题
3. **合并（Combine）**：合并子问题的解

归并排序和快速排序都遵循这个范式，但侧重不同：

| 算法 | 分解 | 解决 | 合并 | 主要工作 |
|------|------|------|------|----------|
| 归并排序 | 平均分割 | 递归排序 | 合并两个有序数组 | 合并 Θ(n) |
| 快速排序 | 按 pivot 分割 | 递归排序 | 无需合并 | 分割 Θ(n) |

你看，归并排序的工作在"合并"阶段，快排的工作在"分割"阶段。两者都是 Θ(n lg n)，但思路截然不同。

## 归并排序：稳定、可预测

### 算法描述

```
MERGE-SORT(A, p, r):
    if p < r:
        q = (p + r) // 2
        MERGE-SORT(A, p, q)
        MERGE-SORT(A, q+1, r)
        MERGE(A, p, q, r)
```

核心是 MERGE 操作：合并两个有序数组。

```
MERGE(A, p, q, r):
    // A[p..q] 和 A[q+1..r] 已排序
    创建临时数组 L 和 R
    i = 0, j = 0, k = p
    while i < len(L) and j < len(R):
        if L[i] <= R[j]:
            A[k] = L[i]; i++
        else:
            A[k] = R[j]; j++
        k++
    复制剩余元素
```

### 复杂度分析

```
T(n) = 2T(n/2) + Θ(n)
     = Θ(n lg n)
```

归并排序保证 Θ(n lg n)，不依赖输入分布。这是它的优点——稳定、可预测。

### 循环不变式：证明 MERGE 正确

同学们，让我们用循环不变式来证明 MERGE 的正确性。循环不变式是证明算法正确性的系统化工具，本质上是数学归纳法在程序上的应用。

**不变式**：在 MERGE 的每次迭代开始时，A[p..k-1] 包含了 L[0..i-1] 和 R[0..j-1] 中最小的 (k-p) 个元素，且已排序。

**证明三步**：

1. **初始化**：k = p, i = 0, j = 0。A[p..k-1] 是空集，包含了 L[0..i-1] 和 R[0..j-1] 中最小的 0 个元素。成立。

2. **保持**：假设迭代前不变式成立。迭代时，比较 L[i] 和 R[j]，较小的放入 A[k]。
   - 如果 L[i] ≤ R[j]，A[k] = L[i]，i++，k++
   - 如果 L[i] > R[j]，A[k] = R[j]，j++，k++
   无论哪种情况，A[p..k] 都包含了最小的 (k-p+1) 个元素，且已排序。不变式保持。

3. **终止**：循环结束时，要么 i = len(L)，要么 j = len(R)。剩余元素复制到 A 后，A[p..r] 包含了所有 L 和 R 的元素，且已排序。算法正确。

这就是循环不变式的威力——不需要灵感，只需检查三个条件。

## 快速排序：原地、概率性

### 算法描述

```
QUICKSORT(A, p, r):
    if p < r:
        q = PARTITION(A, p, r)
        QUICKSORT(A, p, q-1)
        QUICKSORT(A, q+1, r)
```

核心是 PARTITION 操作：把数组分成"小于 pivot"和"大于 pivot"两部分。

```
PARTITION(A, p, r):
    x = A[r]  // 选择最后一个元素作为 pivot
    i = p - 1
    for j = p to r-1:
        if A[j] <= x:
            i++
            swap A[i] and A[j]
    swap A[i+1] and A[r]
    return i + 1
```

### 图解说明

```
初始：[3, 7, 2, 9, 5, 1, 8]  pivot = 8

j=0: 3 ≤ 8 → i++, swap → [3, |7, 2, 9, 5, 1, 8]
j=1: 7 ≤ 8 → i++, swap → [3, 7, |2, 9, 5, 1, 8]
j=2: 2 ≤ 8 → i++, swap → [3, 7, 2, |9, 5, 1, 8]
j=3: 9 > 8 → 无操作
j=4: 5 ≤ 8 → i++, swap → [3, 7, 2, 5, |9, 1, 8]
j=5: 1 ≤ 8 → i++, swap → [3, 7, 2, 5, 1, |9, 8]

最后：swap pivot → [3, 7, 2, 5, 1, 8, |9]

分割结果：pivot 左边都 ≤ 8，右边都 > 8
```

### 循环不变式：证明 PARTITION 正确

**不变式**：每次迭代开始时：
- A[p..i] 中的所有元素 ≤ pivot
- A[i+1..j-1] 中的所有元素 > pivot
- A[j..r-1] 是未处理元素
- A[r] 是 pivot

**证明三步**：

1. **初始化**：i = p-1, j = p。A[p..i] 和 A[i+1..j-1] 都是空集。A[j..r-1] 是 A[p..r-1]。不变式成立。

2. **保持**：
   - 如果 A[j] ≤ pivot：i++, swap A[i] 和 A[j]。A[j] 进入 ≤ pivot 区域。
   - 如果 A[j] > pivot：不操作。A[j] 留在 > pivot 区域。
   无论哪种情况，不变式保持。

3. **终止**：j = r 时，swap A[i+1] 和 A[r]。pivot 进入正确位置。A[p..i] ≤ pivot，A[i+2..r] > pivot。算法正确。

## 快排期望分析：一个概率的洞见

这是概率分析的经典范例。让我们用指示器随机变量来分析快排的期望比较次数。

### 定义指示器随机变量

```
X_{ij} = 1  如果元素 i 和 j 在快排中被比较
        = 0  否则
```

总比较次数 X = Σ_{1 ≤ i < j ≤ n} X_{ij}

### 关键洞见

同学们，仔细想想：两个元素 i 和 j 被比较的唯一条件是什么？

答案是：**它们中第一个被选为 pivot 的元素**。

为什么？如果 i 先被选为 pivot，j 要和 i 比较（决定去左边还是右边）；如果 j 先被选为 pivot，i 要和 j 比较；如果两者都没被先选为 pivot，它们会被分割到不同子数组，不再有机会比较。

### 概率计算

在 i 和 j 之间的元素（包括 i 和 j）共 j - i + 1 个。第一个被选为 pivot 的概率是 2/(j-i+1)（因为 i 或 j 都可能先被选）。

```
E[X] = Σ_{i<j} E[X_{ij}]
     = Σ_{i<j} 2/(j-i+1)
     ≈ 2n Σ_{k=2}^{n} 1/k
     ≈ 2n × ln n
     = O(n lg n)
```

**结论**：快排期望比较次数 = O(n lg n)。

这个分析告诉我们什么？

快排的性能不是"运气"决定的，而是可以用概率精确计算。虽然最坏情况是 Θ(n²)，但期望 Θ(n lg n) 是有保证的——不是靠运气，是靠数学。

## 确定性 vs 随机化：哪个更好？

### 确定性快排

- pivot 选择固定（如第一个、最后一个、中间）
- 最坏 Θ(n²)，期望 Θ(n lg n)
- **依赖输入分布**
- 可以被构造输入攻击

### 随机化快排

- pivot 随机选择
- 最坏 Θ(n²)，期望 Θ(n lg n)
- **不依赖输入分布**
- 无法构造确定性退化输入

| 特性 | 确定性快排 | 随机化快排 |
|------|-----------|-----------|
| 最坏复杂度 | Θ(n²) | Θ(n²) |
| 期望复杂度 | Θ(n lg n) | Θ(n lg n) |
| 依赖输入分布 | ✓ 是 | ✗ 否 |
| 可被攻击 | ✓ 是 | ✗ 否 |

**关键洞见**：随机化快排消除的是对**输入分布**的依赖，不是对**随机数生成器**的依赖。它仍然依赖 RNG 的质量，但这种依赖在实践中可控——比依赖用户输入的分布更可控。

这就是概率思维的核心：**有时候，主动引入随机性，反而比追求确定性更稳定**。
