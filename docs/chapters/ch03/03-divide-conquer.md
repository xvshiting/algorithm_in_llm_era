# 3.3 分治排序：归并与快排

## 学习目标

学完这一节，你能：

---

## War Story：McIlroy 的杀手序列

1999年，Doug McIlroy 发现一个致命问题：某些快排实现可以被"杀手序列"攻击。

他构造了一个特殊输入，使得 qsort() 从 Θ(n lg n) 退化到 Θ(n²)。攻击原理：利用确定性快排的 pivot 选择策略（如"选第一个元素"），构造一个总是选到最小或最大的输入。

**教训**：确定性快排依赖于输入分布。要消除这种依赖，必须引入随机化。

---

## 分治范式

分治策略三步骤：

1. **分解（Divide）**：把问题分成若干子问题
2. **解决（Conquer）**：递归解决子问题
3. **合并（Combine）**：合并子问题的解

归并排序和快速排序都遵循这个范式，但侧重不同：

| 算法 | 分解 | 解决 | 合并 | 主要工作 |
|------|------|------|------|----------|
| 归并排序 | 平均分割 | 递归排序 | 合并两个有序数组 | 合并 Θ(n) |
| 快速排序 | 按 pivot 分割 | 递归排序 | 无需合并 | 分割 Θ(n) |

---

## 归并排序

**算法描述**：

```
MERGE-SORT(A, p, r):
    if p < r:
        q = (p + r) // 2
        MERGE-SORT(A, p, q)
        MERGE-SORT(A, q+1, r)
        MERGE(A, p, q, r)
```

**MERGE 操作**：合并两个有序数组

```
MERGE(A, p, q, r):
    // A[p..q] 和 A[q+1..r] 已排序
    创建临时数组 L 和 R
    i = 0, j = 0, k = p
    while i < len(L) and j < len(R):
        if L[i] <= R[j]:
            A[k] = L[i]; i++
        else:
            A[k] = R[j]; j++
        k++
    复制剩余元素
```

**复杂度分析**：

```
T(n) = 2T(n/2) + Θ(n)
     = Θ(n lg n)
```

归并排序保证 Θ(n lg n)，不依赖输入分布。

## 快速排序

**算法描述**：

```
QUICKSORT(A, p, r):
    if p < r:
        q = PARTITION(A, p, r)
        QUICKSORT(A, p, q-1)
        QUICKSORT(A, q+1, r)
```

**PARTITION 操作**：

```
PARTITION(A, p, r):
    x = A[r]  // 选择最后一个元素作为 pivot
    i = p - 1
    for j = p to r-1:
        if A[j] <= x:
            i++
            swap A[i] and A[j]
    swap A[i+1] and A[r]
    return i + 1
```

**图解说明**：

```
初始：[3, 7, 2, 9, 5, 1, 8]  pivot = 8

j=0: 3 ≤ 8 → i++, swap → [3, |7, 2, 9, 5, 1, 8]
j=1: 7 ≤ 8 → i++, swap → [3, 7, |2, 9, 5, 1, 8]
j=2: 2 ≤ 8 → i++, swap → [3, 7, 2, |9, 5, 1, 8]
j=3: 9 > 8 → 无操作
j=4: 5 ≤ 8 → i++, swap → [3, 7, 2, 5, |9, 1, 8]
j=5: 1 ≤ 8 → i++, swap → [3, 7, 2, 5, 1, |9, 8]

最后：swap pivot → [3, 7, 2, 5, 1, 8, |9]

分割结果：pivot 左边都 ≤ 8，右边都 > 8
```

## 快排期望分析：指示器随机变量

这是概率分析的经典范例。

**定义指示器随机变量**：

```
X_{ij} = 1  如果元素 i 和 j 在快排中被比较
        = 0  否则
```

总比较次数 X = Σ_{1 ≤ i < j ≤ n} X_{ij}

**关键洞察**：两个元素 i 和 j 被比较的唯一条件是：它们中**第一个被选为 pivot 的元素**。

为什么？如果 i 先被选为 pivot，j 要和 i 比较（决定去左还是右）；如果 j 先被选为 pivot，i 要和 j 比较；如果两者都没被先选为 pivot，它们会被分割到不同子数组，不再有机会比较。

**概率计算**：

在 i 和 j 之间的元素（包括 i 和 j）共 j - i + 1 个。第一个被选为 pivot 的概率是 1/(j-i+1)。

```
P[X_{ij} = 1] = 2/(j-i+1)  // i 或 j 先被选
```

**期望计算**：

```
E[X] = Σ_{i<j} E[X_{ij}]
     = Σ_{i<j} 2/(j-i+1)
     = Σ_{k=2}^{n} Σ_{j-i=k} 2/k
     = Σ_{k=2}^{n} (n-k+1) × 2/k
     ≈ 2n Σ_{k=2}^{n} 1/k
     ≈ 2n × ln n
     = O(n lg n)
```

**结论**：快排期望比较次数 = O(n lg n)。

## 确定性 vs 随机化快排

**确定性快排**：
- pivot 选择固定（如第一个、最后一个、中间）
- 最坏 Θ(n²)，期望 Θ(n lg n)
- **依赖输入分布**
- McIlroy 杀手序列可以构造退化输入

**随机化快排**：
- pivot 随机选择
- 最坏 Θ(n²)，期望 Θ(n lg n)
- **不依赖输入分布**
- 无法构造确定性退化输入

**关键区别**：

| 特性 | 确定性快排 | 随机化快排 |
|------|-----------|-----------|
| 最坏复杂度 | Θ(n²) | Θ(n²) |
| 期望复杂度 | Θ(n lg n) | Θ(n lg n) |
| 依赖输入分布 | ✓ 是 | ✗ 否 |
| 可被攻击 | ✓ 是 | ✗ 否 |

随机化快排消除的是对**输入分布**的依赖，不是对**RNG**的依赖。它仍然依赖随机数生成器的质量，但这种依赖在实践中可控。

## 练习

### 层次一（基础）

1. **PARTITION 正确性**：用循环不变式证明 PARTITION 的正确性。

2. **期望计算**：对 n = 8，计算快排期望比较次数的精确值。

3. **杀手序列构造**：对"选第一个元素作为 pivot"的快排，构造一个使其退化的输入。

### 层次二（LLM 协同）

4. **McIlroy 攻击**：让 LLM 实现一个确定性快排（选第一个元素作为 pivot），构造 McIlroy 杀手序列使其退化为 Θ(n²)。

5. **随机化验证**：让 LLM 实现随机化快排，验证不同输入下运行时间的稳定性。

### 层次三（Agent 设计）

6. **快排安全检测 Skill**：设计一个 Skill，检测快排实现是否可以被构造输入攻击。
