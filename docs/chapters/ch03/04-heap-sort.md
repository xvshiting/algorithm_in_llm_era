# 3.4 堆排序

## 学习目标

学完这一节，你能：

- 理解堆的结构性质和堆序性质
- 分析堆排序的复杂度
- 理解建堆线性时间的证明
- 区分堆排序与其他排序的特点

---

## War Story：为什么选择堆排序？

2015年，某嵌入式系统需要排序功能。团队有两个选择：
- 快排：期望快，但最坏 Θ(n²)，可能触发超时
- 归并排序：稳定 Θ(n lg n)，但需要 Θ(n) 额外空间，内存不足

团队选择了堆排序：原地 Θ(n lg n)，最坏保证，内存友好。

**教训**：不同场景需要不同权衡。堆排序在"原地 + 最坏保证"组合上有独特优势。

---

## 堆的基本概念

**堆**是满足两个性质的数组：

1. **结构性质**：完全二叉树（除最后一层外每层满，最后一层从左到右填充）
2. **堆序性质**：每个节点 ≥ 其子节点（最大堆）

**数组表示**：

```
树形表示：
        16
       /  \
      14   10
     / \   / \
    8  7  9  3
   / \
  2  4

数组表示：[16, 14, 10, 8, 7, 9, 3, 2, 4]
索引：     0   1   2  3  4  5  6  7  8

父节点：parent(i) = (i-1) // 2
左子节点：left(i) = 2i + 1
右子节点：right(i) = 2i + 2
```

---

## 堆的关键操作

**MAX-HEAPIFY(A, i)**：维护堆序性质

```
MAX-HEAPIFY(A, i):
    l = left(i)
    r = right(i)
    largest = i
    if l < len(A) and A[l] > A[largest]:
        largest = l
    if r < len(A) and A[r] > A[largest]:
        largest = r
    if largest != i:
        swap A[i] and A[largest]
        MAX-HEAPIFY(A, largest)
```

**图解说明**：

```
调用 MAX-HEAPIFY(A, 2)：
初始：[16, 14, 10, 8, 7, 9, 3]
           A[2]=10, left=9, right=3

比较：largest = 2 (10)
     A[5]=9 < 10, 不更新
     A[6]=3 < 10, 不更新

结果：不变（已经满足堆序）
```

**复杂度**：Θ(h)，h 是节点高度。最坏 Θ(lg n)。

---

## 建堆：BUILD-MAX-HEAP

**算法**：

```
BUILD-MAX-HEAP(A):
    for i = len(A)//2 down to 0:
        MAX-HEAPIFY(A, i)
```

**为什么从 len(A)//2 开始？**

索引 ≥ len(A)//2 的节点都是叶子节点，不需要 heapify。

**线性时间证明**：

关键洞察：不同高度的节点数量不同。

```
高度 h 的节点数量 ≤ ⌈n/2^{h+1}⌉

总时间 = Σ_{h=0}^{⌊lg n⌋} ⌈n/2^{h+1}⌉ × O(h)
       ≤ Σ_{h=0}^{∞} n/2^{h+1} × O(h)
       = O(n Σ_{h=0}^{∞} h/2^{h+1})
       = O(n × 2)
       = O(n)
```

**结论**：建堆时间是 Θ(n)，不是 Θ(n lg n)！

---

## 堆排序：HEAPSORT

**算法**：

```
HEAPSORT(A):
    BUILD-MAX-HEAP(A)  // Θ(n)
    for i = len(A)-1 down to 1:
        swap A[0] and A[i]  // 最大元素放到末尾
        MAX-HEAPIFY(A[0..i-1], 0)  // Θ(lg i)
```

**图解说明**：

```
初始：[16, 14, 10, 8, 7, 9, 3, 2, 4]  已建堆

i=8: swap A[0]和A[8] → [4, 14, 10, 8, 7, 9, 3, 2 | 16]
     heapify → [14, 8, 10, 4, 7, 9, 3, 2 | 16]

i=7: swap → [2, 8, 10, 4, 7, 9, 3 | 14, 16]
     heapify → [10, 8, 9, 4, 7, 2, 3 | 14, 16]

...继续...

最终：[2, 3, 4, 7, 8, 9, 10, 14, 16]  已排序
```

**复杂度**：

```
T(n) = Θ(n) + Σ_{i=1}^{n-1} Θ(lg i)
     = Θ(n) + Θ(n lg n)
     = Θ(n lg n)
```

## 堆排序的特点

| 特性 | 堆排序 | 快排 | 归并排序 |
|------|--------|------|----------|
| **时间** | Θ(n lg n) 最坏保证 | Θ(n lg n) 期望，Θ(n²) 最坏 | Θ(n lg n) 最坏保证 |
| **空间** | Θ(1) 原地 | Θ(lg n) 递归栈 | Θ(n) 额外数组 |
| **稳定** | ✗ 不稳定 | ✗ 不稳定 | ✓ 稳定 |
| **适用场景** | 内存受限、需要最坏保证 | 一般场景、缓存友好 | 大数据、需要稳定 |

5. **优化探索**：让 LLM 探索堆排序的优化策略（如 Floyd 堆构造），比较性能。
