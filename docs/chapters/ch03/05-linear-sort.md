## War Story：错误地突破下界

回到开篇的故事：工程师小张被告知"计数排序 O(n) 更快"，直接应用，系统崩溃。

计数排序的前提是：键值范围 k ≤ n 或 k = O(n)。

搜索系统的键是浮点数相关性得分，范围远超 n。Cursor 不知道这个前提，小张没验证——两错。

**教训**：线性排序不是"无条件更快"，而是"在特定条件下更快"。知道条件是判断能力。

---

## 计数排序

**适用条件**：键是整数，范围 k ≤ n 或 k = O(n)

**算法思想**：不比较，直接"计数定位"

```
COUNTING-SORT(A, k):
    // A 中元素范围 [0, k]
    创建计数数组 C[0..k]
    初始化 C 全为 0
    
    // 计数阶段
    for each x in A:
        C[x]++
    
    // 累计阶段（变成"第几个位置"）
    for i = 1 to k:
        C[i] += C[i-1]
    
    // 输出阶段（从后往前，保证稳定）
    创建输出数组 B[0..n-1]
    for i = n-1 down to 0:
        B[C[A[i]]-1] = A[i]
        C[A[i]]--
    
    return B
```

**图解说明**：

```
输入：A = [2, 5, 3, 0, 2, 3, 0, 3]，k = 5

计数：C = [2, 0, 2, 3, 0, 1]  // 0出现2次，2出现2次，...
累计：C = [2, 2, 4, 7, 7, 8]  // 0-1放前2位，2放3-4位，...

输出（从后往前）：
i=7: A[7]=3 → B[C[3]-1]=B[6]=3, C[3]=6
i=6: A[6]=0 → B[C[0]-1]=B[1]=0, C[0]=1
...

最终：B = [0, 0, 2, 2, 3, 3, 3, 5]
```

**复杂度**：Θ(n + k)

**关键问题**：如果 k >> n，比如 n = 1000 但 k = 10^9，计数排序不可用。

## 基数排序

**适用条件**：键可以分解为 d 位，每位的范围有限

**算法思想**：从最低位到最高位，逐位稳定排序

```
RADIX-SORT(A, d):
    for i = 1 to d:
        用稳定排序（如计数排序）按第 i 位排序
```

**图解说明**：

```
输入：[329, 457, 657, 839, 436, 720, 355]

第1位（个位）排序：[720, 355, 436, 457, 657, 329, 839]
第2位（十位）排序：[720, 355, 436, 457, 657, 329, 839]  // 稳定保持顺序
第3位（百位）排序：[329, 355, 436, 457, 657, 720, 839]

注意：必须从低位到高位，且必须用稳定排序
```

**复杂度**：Θ(d(n + k))，k 是每位的范围

**为什么必须从低位到高位？**

低位排序后，低位已经有序。高位排序时，如果高位相同，稳定排序会保持低位顺序。

**为什么必须稳定？**

如果不稳定，高位相同的元素可能打乱低位顺序。

## 桶排序

**适用条件**：输入均匀分布在 [0, 1)

**算法思想**：把区间分成 n 个桶，每个桶期望放 1 个元素

```
BUCKET-SORT(A):
    创建 n 个空桶 B[0..n-1]
    for i = 0 to n-1:
        把 A[i] 放入桶 B[⌊n × A[i]⌋]
    for each bucket B[i]:
        用插入排序对 B[i] 排序
    串联所有桶
```

**期望复杂度分析**：

关键：每个桶期望放多少元素？

设 n_i 是第 i 个桶的元素数。均匀分布下：

```
E[n_i] = 1
E[n_i²] = 2 - 1/n  // 为什么？
```

计算：

```
P[元素落入桶 i] = 1/n

n_i = Σ_{j=1}^{n} I_j  // I_j 是指示器变量

E[n_i²] = E[(Σ I_j)²] = Σ E[I_j] + Σ_{j≠k} E[I_j I_k]
         = n × 1/n + n(n-1) × 1/n²
         = 1 + (n-1)/n
         = 2 - 1/n
```

插入排序 Θ(n_i²)，所以：

```
E[总时间] = n × Θ(E[n_i²]) = n × Θ(2 - 1/n) = Θ(n)
```

**结论**：均匀分布下，桶排序期望 Θ(n)。

## 三种线性排序对比

| 算法 | 时间 | 空间 | 稳定 | 适用条件 |
|------|------|------|------|----------|
| **计数排序** | Θ(n + k) | Θ(k) | ✓ | 整数键，范围 k ≤ n |
| **基数排序** | Θ(d(n + k)) | Θ(n + k) | ✓ | 可分解为 d 位 |
| **桶排序** | Θ(n) 期望 | Θ(n) | ✓ | 均匀分布浮点数 |

**选择指南**：

| 场景 | 推荐算法 |
|------|----------|
| 年龄排序（0-150） | 计数排序 |
| 电话号码排序 | 基数排序 |
| 随机生成的成绩排序 | 桶排序 |
| 字符串按字典序排序 | 基数排序（字符为位） |
| 一般整数排序 | 快排/堆排序（k 可能太大） |

## 练习

### 层次一（基础）

1. **计数排序验证**：对数组 [1, 4, 1, 2, 7, 5, 2]，手动执行计数排序。

2. **基数排序执行**：对 [170, 45, 75, 90, 802, 24, 2, 66]，执行基数排序（按十进制位）。

3. **桶排序期望**：计算 n = 10 时，桶排序的期望时间。

### 层次二（LLM 协同）

4. **条件判断**：给 LLM 一个排序场景（键类型、范围、数据量），让它判断能否用线性排序。

5. **对比实验**：让 LLM 实现计数排序和快排，对比不同 k 值下的性能。

### 层次三（Agent 设计）

6. **自动选择 Skill**：设计一个 Skill，根据输入特征自动选择线性排序或比较排序。
