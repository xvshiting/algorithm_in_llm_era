## War Story：为什么哈希是随机化算法？

2018年，某团队设计缓存系统。工程师小李用哈希表存储 URL 缓存，遇到问题：某些 URL 总是"碰"在一起（哈希冲突）。

小李想：是不是哈希函数不够好？

同事指出："不是哈希函数的问题，是哈希的本质。哈希是用概率换取性能——期望 O(1)，最坏 Θ(n)。这不是缺陷，是设计。"

**Skiena 的视角转换**：

> 哈希不是数据结构技巧，是随机化算法。

传统视角：哈希表是"快速查找的数据结构"
Skiena 视角：哈希是"用随机性换取期望性能的算法"

这个视角转换，是理解 LLM 采样策略的关键基础。

## Balls-and-Bins 模型

哈希冲突的本质是 balls-and-bins 问题：把 n 个球随机扔进 m 个桶。

**核心问题**：最多会有多少个球落入同一个桶？

### 1. 单桶期望负载

```
E[每个桶的球数] = n/m  // 简单的期望
```

### 2. 最大负载

**定理**：均匀哈希下，最大负载期望 ≈ ln n / ln ln n（当 m = n）

这不是直觉结果！即使 n 个球扔进 n 个桶，最大负载不是常数，而是缓慢增长的。

**直觉**：
- 概率最大桶恰好有 k 个球 = (n choose k) × (1/n)^k × (1-1/n)^{n-k}
- 要使得概率 > 0，需要 k ≈ ln n / ln ln n

### 3. 冲突期望

```
期望冲突次数 = n(n-1)/2m
```

**证明**：

```
每对球的冲突概率 = 1/m
总对数 = n(n-1)/2
期望冲突 = n(n-1)/2m
```

**应用**：设计哈希表时，选择 m 使得期望冲突可控。

## 全域散列

**问题**：确定性哈希函数可能被"攻击"——构造冲突输入

**解决方案**：全域散列（Universal Hashing）

**定义**：哈希函数族 H 是全域的，如果：

```
P_{h∈H}[h(x) = h(y)] ≤ 1/m  // 对任意 x ≠ y
```

**定理**：全域散列族的期望冲突次数 ≤ n(n-1)/2m

**构造方法**：

```
H = {h_{a,b}(x) = ((ax + b) mod p) mod m}
其中：
- p 是大于键值范围的质数
- a ∈ {1, ..., p-1}
- b ∈ {0, ..., p-1}
```

**为什么是全域的？**

```
P[h(x) = h(y)] = P[((ax + b) mod p) mod m = ((ay + b) mod p) mod m]

设 r_x = (ax + b) mod p, r_y = (ay + b) mod p
r_x ≠ r_y（因为 a ≠ 0 且 x ≠ y）

r_x - r_y = a(x - y) mod p ≠ 0

要让 ((r_x mod m) = (r_y mod m))，需要 r_x ≡ r_y (mod m)
即 r_x - r_y ≡ 0 (mod m)

概率 ≤ 1/m（因为 r_x - r_y 在 mod p 下均匀）
```

**应用**：随机选择哈希函数，消除对输入分布的依赖。

## Bloom Filter：假阳性可接受

**问题**：快速判断元素"是否在集合中"，但允许假阳性

**设计**：

```
Bloom Filter：
- 位数组 B[0..m-1]，初始全 0
- k 个哈希函数 h_1, ..., h_k

插入元素 x：
    for i = 1 to k:
        B[h_i(x) mod m] = 1

查询元素 x：
    return B[h_1(x)] ∧ B[h_2(x)] ∧ ... ∧ B[h_k(x)]
```

**图解说明**：

```
m=10 位数组，k=3 哈希函数

插入 "apple"：
h_1("apple") = 3, h_2("apple") = 7, h_3("apple") = 9
B = [0,0,0,1,0,0,0,1,0,1]

插入 "banana"：
h_1("banana") = 1, h_2("banana") = 4, h_3("banana") = 9
B = [0,1,0,1,1,0,0,1,0,1]

查询 "apple"：
B[3]=1, B[7]=1, B[9]=1 → ✓ 可能存在

查询 "cherry"（假设哈希值 2, 5, 8）：
B[2]=0 → ✗ 不存在（确定性）

查询 "orange"（假设哈希值 3, 4, 7）：
B[3]=1, B[4]=1, B[7]=1 → ✓ 可能存在（假阳性！）
```

**假阳性概率**：

```
P[假阳性] = P[所有 k 个位置都是 1]
         ≈ (1 - e^{-kn/m})^k
```

**假阴性概率**：0（确定性）

**为什么假阴性是 0？**

插入时把所有 k 个位置设为 1，查询时检查所有 k 个位置。如果任何位置是 0，说明元素从未插入。

**设计哲学**：

```
┌─────────────────────────────────────────────────────────────────────┐
│  Bloom Filter 的核心洞察                                             │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  假阳性可接受 → 多候选验证后排除                                    │
│  假阴性不可接受 → 会漏掉真正存在                                    │
│                                                                     │
│  类比：                                                              │
│  ├── 搜索引擎：假阳性（多结果）可接受，假阴性（漏结果）不可接受    │
│  ├── LLM 采样：假阳性（多候选）可接受，假阴性（漏最佳）不可接受    │
│  ├── 缓存系统：假阳性（缓存不存在）可接受，假阴性（缓存存在）不可 │
│  └──────────────────────────────────────────────────────────────────│
│                                                                     │
│  这是"概率换取效率"的设计原则                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## MinHash：相似度的概率估计

**问题**：快速估计两个集合的相似度（Jaccard 相似度）

```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

**MinHash 方法**：

```
h_min(S) = argmin_{x∈S} h(x)  // 哈希值最小的元素

定理：P[h_min(A) = h_min(B)] = Jaccard(A, B)
```

**直觉**：

- A ∪ B 中，哈希值最小的元素落入 A ∩ B 的概率 = |A ∩ B| / |A ∪ B|
- 每个元素落入 A ∩ B 的概率相等（哈希均匀）

**应用**：

```
估计 Jaccard(A, B)：
- 使用 k 个哈希函数
- 计算 h_min(A) 和 h_min(B) 对每个哈希函数
- 相等的比例 ≈ Jaccard(A, B)

复杂度：O(k) 而不是 O(|A| + |B|)
```

**图解说明**：

```
集合 A = {a, b, c}, B = {b, c, d}

哈希函数 h_1: h(a)=5, h(b)=2, h(c)=7, h(d)=1
h_min(A) = b (最小哈希值 2)
h_min(B) = d (最小哈希值 1)
不相等 → 不贡献

哈希函数 h_2: h(a)=3, h(b)=6, h(c)=1, h(d)=8
h_min(A) = c (最小哈希值 1)
h_min(B) = c (最小哈希值 1)
相等 → 贡献 1

...k 个哈希函数...

相等比例 ≈ 2/4 = 0.5（Jaccard(A,B) = |{b,c}|/|{a,b,c,d}| = 2/4）
```

## 完美散列

**问题**：静态集合（元素不动态插入删除），需要 O(1) 最坏查询

**二级方案**：

```
第一级：哈希函数 h_1 把 n 个元素分到 m 个桶
第二级：每个桶有自己的哈希函数 h_2 和小哈希表

关键：每个桶的大小 = n_i²（无冲突）
```

**定理**：E[Σ n_i²] < 2n（当 m ≈ n）

**证明**：

```
每个桶的期望大小 = 1
方差 = E[n_i²] - E[n_i]^2

如果哈希是全域的：
E[n_i²] = E[n_i] + E[n_i]^2 - E[n_i] = 1 + 1 - 1 = 1（单桶）
Σ E[n_i²] < 2n（总和）
```

**应用**：编译器符号表、静态查找表

## 哈希设计原则总结

| 原则 | 含义 | 应用 |
|------|------|------|
| **随机性是设计** | 用概率换取期望性能 | 全域散列、随机化快排 |
| **假阳性可接受** | 多候选后验证排除 | Bloom Filter、LLM Top-k |
| **假阴性不可接受** | 会漏掉真正存在 | Bloom Filter 保证 |
| **概率估计代替精确计算** | 用采样估计全局量 | MinHash、LLM 采样 |
| **最坏可控** | 静态集合可以 O(1) 最坏 | 完美散列 |

### 层次一（基础）

1. **冲突期望计算**：n = 1000, m = 100，计算期望冲突次数。

2. **Bloom Filter 假阳性率**：m = 1000, n = 100, k = 5，计算假阳性概率。

3. **MinHash 验证**：集合 A = {1,2,3}, B = {2,3,4}，Jaccard = 2/4 = 0.5，用 MinHash 验证。

4. **全域散列分析**：让 LLM 解释全域散列的构造方法，分析为什么 P[h(x)=h(y)] ≤ 1/m。

6. **哈希策略 Skill**：设计一个 Skill，根据应用场景选择哈希策略（链式、开放寻址、Bloom Filter、MinHash）。
