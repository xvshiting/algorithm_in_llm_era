# 第5章 组合搜索——如何系统地探索巨大空间

## 本章核心叙事

这章的核心概念和 LLM 推理是**同构的**：贪心解码 = 局部搜索，思维链 = DFS，思维树 = 回溯+剪枝，思维图 = A*。不是外在映射，而是同一思想的不同实例。理解搜索策略就是理解 LLM 如何"思考"。

## 5.1 回溯搜索——解空间上的 DFS

解建模为向量 a = (a₁, ..., aₙ)，回溯是系统性的 DFS。为什么选 DFS 不选 BFS：空间 O(高度) vs O(宽度)。回溯五子程序框架：is_a_solution、constructCandidates、process_solution、make_move、unmake_move。

## 5.2 搜索剪枝策略

最受约束优先。前瞻剪枝。Sudoku 从 1,904,832 步到 48 步。对称性剪枝（TSP 固定起点省 n 倍）。

## 5.3 最佳优先搜索与 A*

分支限界用优先队列管理部分解。A* 加上可容许下界估计保证最优性——但空间换时间（优先队列峰值 vs 回溯栈深度）。

## 5.4 启发式搜索——从局部搜索到模拟退火

局部搜索的局部最优陷阱。"让启发式更复杂不一定更好"——贪心度数顶点覆盖 Θ(log n) 比简单匹配 2-近似更差。SA 的 e^{ΔE/kT} 接受劣转移，dying heartbeat 收敛。代价函数必须给出部分信用。

## 5.5 从搜索策略到 LLM 推理策略

| 搜索策略 | LLM 推理对应 |
|----------|-------------|
| 贪心解码 | 每步选最高概率 token |
| 思维链（CoT） | DFS——线性展开推理路径 |
| 思维树（ToT） | 回溯+剪枝——多分支探索 |
| 思维图（GoT） | A*——评估每条推理路径的前景 |

**关键洞察**：理解搜索策略的局限性（局部搜索陷入局部最优、SA 需要好的代价函数、A* 需要可容许下界）直接帮助理解 LLM 推理的局限性。贪心解码会陷入重复，思维树需要好的剪枝条件，思维图需要好的启发式函数——这些都是同一问题的不同面。

## 5.6 练习

**层次一（基础）**：
1. 用回溯框架实现子集生成和排列生成
2. 实现 Sudoku 求解器，比较有剪枝和无剪枝的步骤数差异
3. 给出一个 TSP 小实例，手动追踪 A* 的优先队列状态

**层次二（LLM 协同）**：
4. 让 LLM 分别用贪心、回溯、SA 求解同一组合优化问题，对比解的质量和运行时间
5. 构造一个让 LLM 的贪心策略陷入局部最优的问题实例

**层次三（Agent 设计）**：
6. 设计一个搜索策略选择 Agent：输入问题特征（解空间大小、约束强度、是否需要最优解），自动选择回溯/剪枝/A*/SA 并调参
