# 第6章 动态规划——缓存与优化的艺术

## 本章核心叙事

这章的核心逻辑是**人负责发现问题结构，机器负责填充表格**。DP 记忆化 ↔ KV-cache 是同构的——存储已计算的公共前缀。但识别最优子结构需要人的判断，这是机器做不了的。

## 6.1 从递归到动态规划——三阶递进

Fibonacci：朴素递归 O(1.6^n) → 记忆化 O(n) → 自底向上 O(n)（可控空间至 O(1)）。缓存只在参数空间小且值被多次复用时有效——对 quicksort 和回溯无效。

**子问题图**将 DP 连接到图论：子问题形成 DAG，按反拓扑序求值。运行时间 ≈ O(|V| + |E|)。

## 6.2 DP 四步法与最优子结构

CLRS 的四步法：(1) 刻画最优子结构 → (2) 递归定义 → (3) 自底向上计算 → (4) 构造最优解。

**最优子结构的形式化刻画**和**剪贴证明**：假设子问题的解不是最优的，剪下替换——矛盾。这是比 Skiena 三步法更严格的正确性保证。

**最长简单路径没有最优子结构**——两条最长子路径可能共享顶点，拼接后不是简单路径。0-1 背包有最优子结构但需 DP，分数背包有且可贪心——同一问题结构，不同可处理性。

## 6.3 编辑距离——可插拔代价的统一框架

Skiena 的核心贡献：一个 DP 表 + 四个可插拔桩函数解决一族问题（编辑距离、子串匹配、LCS、最大单调子序列）。路径重构用 parent 指针从目标回溯到原点。

## 6.4 更多 DP 问题

**钢条切割**：两种递推揭示不同子问题结构——自顶向下 r_n = max(p_i + r_{n-i}) vs 自底向上。

**矩阵链乘**：m[i,j] = min_{i≤k<j} {m[i,k] + m[k+1,j] + p_{i-1}p_kp_j}，Θ(n³)。Knuth 优化利用最优分割点单调性将 Θ(n³) 降到 Θ(n²)。**例题**：矩阵链 A₁(30×35)·A₂(35×15)·A₃(15×5)·A₄(5×10)，最优括号化为 ((A₁A₂)(A₃A₄))，总乘法次数 15125 而非按序的 8375+78750。

**LCS**：编辑距离框架的特例——c[i,j] = {c[i-1,j-1]+1 (x_i=y_j), max(c[i-1,j],c[i,j-1]) (otherwise)}。

**最优 BST**：给定搜索概率，构造期望搜索代价最小的 BST。e[i,j] = min_{i≤k≤j} {e[i,k-1]+e[k+1,j]+w(i,j)}，Θ(n³)。最优 BST 不是高度平衡树——概率分布决定形状。

## 6.5 DP 的正确性与效率条件

最优子结构 + 重叠子问题 + 多项式参数空间。最长简单路径失败——与路径历史有关。TSP 的 O(2^n) 子集状态使 DP 退化为指数。

**有序约束是 DP 效率的关键**：Prolog 合一并的 trie 优化中，规则顺序保留使 DP 可解（O(n²)），无序则是 NP 完全。

## 6.6 练习

**层次一（基础）**：
1. 用剪贴证明论证矩阵链乘法的最优子结构
2. 实现编辑距离的统一框架，通过修改桩函数分别解决 LCS 和子串匹配
3. 证明最长简单路径缺乏最优子结构

**层次二（LLM 协同）**：
4. 让 LLM 为一个新问题给出 DP 递推式，你用最优子结构的剪贴证明验证其正确性
5. 让 LLM 生成贪心解和 DP 解，在同一问题（如书本分页）上对比——贪心可能"很好"但 DP 始终最优

**层次三（Agent 设计）**：
6. 设计 DP 识别器 Skill：输入问题描述，自动判断是否适合 DP（子问题重叠？最优子结构？多项式参数空间？），生成递推式和实现框架
