# 7.1 图的表示——从地图到邻接表

> **问题情境**：2024年某导航公司发现，同样的地图数据，用不同表示方法存储，内存占用相差10倍。为什么？

---

## 问题情境：地图的存储难题

小明在某导航公司实习，接到一个任务：存储全国道路网络数据。

他写了两种代码：

**方案A（邻接矩阵）**：
```python
# 假设10000个城市
matrix = [[0] * 10000 for _ in range(10000)]  # 10000 x 10000矩阵
```
内存占用：10000² × 4字节 ≈ 400MB

**方案B（邻接表）**：
```python
# 每个城市只存相邻城市
adj = [[] for _ in range(10000)]  # 平均每个城市连接5条路
```
内存占用：10000 × 5 × 8字节 ≈ 400KB

400MB vs 400KB，相差1000倍！

老板问："为什么差异这么大？哪个方案更好？"

小明答不上来。他只知道"邻接表存邻居"，但不知道背后的原理。

**这一节要回答的核心问题**：**如何在计算机中高效存储图结构？空间和时间的权衡是什么？**

---

## 直观思路：地图是什么？

### 图的本质

回到地图：什么是道路网络？

- **路口** = **节点**（Vertex）
- **道路** = **边**（Edge）

这就是**图**（Graph）：节点和边的集合。

但问题是：**如何把"节点和边"存进计算机？**

### 两种直觉

**直觉1：全记录**

把所有可能的连接都记录下来，用一个大表格：

| 路口A | 路口B | 路口C | 路口D |
|-------|-------|-------|-------|
| A→B: 有路 | A→C: 无路 | A→D: 有路 | ... |
| B→A: 有路 | B→C: 有路 | B→D: 无路 | ... |
| ... | ... | ... | ... |

这就是**邻接矩阵**：n×n的表格，记录所有可能的边。

**直觉2：只记邻居**

每个路口只记录"它连接哪些路"：

- 路口A: 连接B、D
- 路口B: 连接A、C
- 路口C: 连接B
- 路口D: 连接A

这就是**邻接表**：每个节点一个列表，只存实际存在的边。

### 关键洞察

小明的问题答案藏在**图的特性**里：

- **道路网络**：稀疏图（每个路口只连接几条路）
  - 10000个路口，实际只有约50000条路（平均每个路口5条）
  - 理论最多10000²=1亿条可能的连接
  - 实际边数/理论最大边数 = 50000/1亿 = 0.0005（极稀疏）

- **邻接矩阵**：记录所有可能连接，包括"无路"
  - 空间：O(n²)，不管实际有多少边
  - 稀疏图浪费大量空间存"无路"

- **邻接表**：只记录实际存在的边
  - 空间：O(n + m)，n是节点数，m是边数
  - 稀疏图节省空间

**选择数据结构要看"图的性格"**——稀疏还是稠密？

## 规范定义：图的形式化

### 图的定义

**定义（图）**：图G = (V, E)由两部分组成：
- V：节点集合（vertices）
- E：边集合（edges），每条边连接两个节点

**图的类型**：
- **无向图**：边没有方向，(u, v)和(v, u)是同一条边
- **有向图**：边有方向，u→v和v→u是不同的边
- **加权图**：每条边有权重（如距离、成本）

### 邻接矩阵

**定义（邻接矩阵）**：对于n个节点的图G = (V, E)，邻接矩阵A是n×n矩阵：

- 无向图：A[i][j] = 1 如果存在边(i, j)，否则0
- 有向图：A[i][j] = 1 如果存在边i→j，否则0
- 加权图：A[i][j] = w 如果边(i, j)权重为w，否则∞（或0）

**空间复杂度**：O(V²)

**时间复杂度**：
- 查询边是否存在：O(1)
- 遍历某节点的所有邻居：O(V)
- 遍历所有边：O(V²)

### 邻接表

**定义（邻接表）**：对于图G = (V, E)，邻接表是数组+链表：

- 数组：长度为V，每个元素对应一个节点
- 链表：节点i的链表存储所有邻居j（及边权重）

**空间复杂度**：O(V + E)

**时间复杂度**：
- 查询边是否存在：O(deg(v))，需要遍历邻居列表
- 遍历某节点的所有邻居：O(deg(v))
- 遍历所有边：O(V + E)

### 边列表

**定义（边列表）**：直接存储所有边，每条边是三元组(u, v, w)。

**适用场景**：
- Kruskal算法：需要按边权重排序
- 边迭代操作频繁

**空间复杂度**：O(E)

**时间复杂度**：
- 查询边：O(E)
- 遍历边：O(E)

## 算法实现：三种表示的代码

### 邻接矩阵实现

```python
class AdjacencyMatrix:
    """邻接矩阵表示"""
    
    def __init__(self, n, directed=False):
        self.n = n
        self.directed = directed
        self.matrix = [[float('inf')] * n for _ in range(n)]
        # 对角线设为inf（自己到自己无边）
        for i in range(n):
            self.matrix[i][i] = float("inf")
    
    def add_edge(self, u, v, weight=1):
        """添加边"""
        self.matrix[u][v] = weight
        if not self.directed:
            self.matrix[v][u] = weight
    
    def has_edge(self, u, v):
        """查询边是否存在"""
        return self.matrix[u][v] != float('inf')
    
    def get_neighbors(self, u):
        """获取邻居"""
        return [(v, self.matrix[u][v]) 
                for v in range(self.n) 
                if self.matrix[u][v] != float('inf') and v != u]
    
    def get_edge_weight(self, u, v):
        """获取边权重"""
        return self.matrix[u][v]
```

### 邻接表实现

```python
class AdjacencyList:
    """邻接表表示"""
    
    def __init__(self, n, directed=False):
        self.n = n
        self.directed = directed
        self.adj = [[] for _ in range(n)]  # 每个节点的邻居列表
    
    def add_edge(self, u, v, weight=1):
        """添加边"""
        self.adj[u].append((v, weight))
        if not self.directed:
            self.adj[v].append((u, weight))
    
    def has_edge(self, u, v):
        """查询边是否存在"""
        for (neighbor, _) in self.adj[u]:
            if neighbor == v:
                return True
        return False
    
    def get_neighbors(self, u):
        """获取邻居"""
        return self.adj[u]
    
    def get_edge_weight(self, u, v):
        """获取边权重"""
        for (neighbor, weight) in self.adj[u]:
            if neighbor == v:
                return weight
        return float('inf')
```

### 边列表实现

```python
class EdgeList:
    """边列表表示"""
    
    def __init__(self, directed=False):
        self.directed = directed
        self.edges = []  # [(u, v, weight)]
        self.nodes = set()
    
    def add_edge(self, u, v, weight=1):
        """添加边"""
        self.edges.append((u, v, weight))
        self.nodes.add(u)
        self.nodes.add(v)
    
    def get_edges_sorted(self):
        """按权重排序的边（用于Kruskal）"""
        return sorted(self.edges, key=lambda e: e[2])
    
    def has_edge(self, u, v):
        """查询边是否存在"""
        for (eu, ev, _) in self.edges:
            if (eu == u and ev == v) or (not self.directed and eu == v and ev == u):
                return True
        return False
```

## 正确性分析：三种表示等价

### 等价性定理

**定理**：邻接矩阵、邻接表、边列表可以相互转换，表示同一个图。

**证明**：
- 邻接矩阵 → 邗接表：遍历矩阵，对每个非零元素(u, v, w)，添加到adj[u]
- 邗接表 → 边列表：遍历每个节点的邻居列表，收集所有边
- 边列表 → 邻接矩阵：对每条边(u, v, w)，设置matrix[u][v] = w

转换后，三种表示包含相同的节点集合和边集合。

### 操作正确性

**查询边**：
- 邻接矩阵：直接访问matrix[u][v]，结果正确
- 邗接表：遍历adj[u]找v，因为添加边时已记录，结果正确

**遍历邻居**：
- 邗接矩阵：遍历matrix[u]的所有列，非inf即为邻居（对角线已设为inf）
- 邗接表：adj[u]直接存储邻居，结果正确

## 复杂度分析：时间空间权衡

### 操作复杂度对比

| 操作 | 邗接矩阵 | 邗接表 | 边列表 |
|------|----------|--------|--------|
| 空间 | O(V²) | O(V+E) | O(E) |
| 添加边 | O(1) | O(1) | O(1) |
| 删除边 | O(1) | O(deg(v)) | O(E) |
| 查询边 | O(1) | O(deg(v)) | O(E) |
| 遍历邻居 | O(V) | O(deg(v)) | O(E) |
| 遍历所有边 | O(V²) | O(V+E) | O(E) |

### 选择策略

**邻接矩阵适用场景**：
1. **稠密图**：边数接近V²（如完全图）
2. **频繁查询边**：需要快速判断"u和v是否连接"
3. **静态图**：边不频繁增删
4. **小规模图**：V较小，V²可接受

**邻接表适用场景**：
1. **稀疏图**：边数远小于V²（如道路网络、社交网络）
2. **频繁遍历邻居**：需要快速获取"u的所有邻居"
3. **动态图**：边频繁增删
4. **大规模图**：V很大，V²不可接受

**边列表适用场景**：
1. **边迭代操作**：需要对所有边排序或遍历（如Kruskal）
2. **边的增删频繁**：但边的查询不频繁
3. **作为辅助表示**：与邻接表配合使用

### 实际案例对比

**案例1：道路网络**
- 10000个城市，50000条路
- 邗接矩阵：400MB
- 邗接表：约400KB
- **选择**：邻接表（稀疏）

**案例2：社交网络**
- 1亿用户，平均每人100好友
- 邗接矩阵：1亿² × 4字节 = 4TB（不可接受）
- 邗接表：1亿 × 100 × 8字节 ≈ 80GB（可接受）
- **选择**：邻接表（极稀疏）

**案例3：棋盘游戏**
- 8×8棋盘，每格最多8邻居
- 邗接矩阵：64 × 64 × 4字节 = 16KB（可接受）
- 邗接表：64 × 8 × 8字节 ≈ 4KB
- **选择**：均可（小规模）

```python
def matrix_to_list(matrix):
    """邻接矩阵转换为邻接表"""
    n = len(matrix)
    adj = [[] for _ in range(n)]
    for u in range(n):
        for v in range(n):
            if matrix[u][v] != 0 and matrix[u][v] != float('inf') and u != v:
                adj[u].append((v, matrix[u][v]))
    return adj
```

## 本节小结

### 核心问题回答

**问题**：如何在计算机中高效存储图结构？

**答案**：
- **邻接矩阵**：存储所有可能的边，适合稠密图和频繁边查询
- **邻接表**：只存储实际存在的边，适合稀疏图和频繁邻居遍历
- **边列表**：直接存储边，适合边迭代操作

### 关键洞察

1. **选择数据结构要看"图的性格"**：稀疏还是稠密？
2. **时间空间权衡**：邻接矩阵查询快但空间大，邻接表空间小但查询慢
3. **三种表示等价**：可以相互转换，表示同一个图
