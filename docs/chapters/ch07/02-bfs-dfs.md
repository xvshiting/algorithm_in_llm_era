# 7.2 BFS 与 DFS——探索网络的两种策略

> **问题情境**：你在一个陌生的城市，想找到最近的地铁站。应该"逐圈搜索"还是"沿着一条路走到底"？

---

## 问题情境：两种探索策略

小明到一个陌生城市出差，想知道最近的地铁站在哪里。

他有两种策略：

**策略A（逐圈搜索）**：
- 先找周边100米的地铁站
- 没有，扩大到200米
- 没有，扩大到300米
- 找到了！

**策略B（沿路走到底）**：
- 选一条路，一直走
- 走到尽头，没找到，回头
- 换另一条路，一直走
- 找到了！

哪个策略更高效？

小明发现：如果只想找"最近"的地铁站，策略A更好（找到第一个就是最近的）。如果想探索"这个区域有哪些地铁站"，策略B更好（能找到所有地铁站）。

**这一节要回答的核心问题**：**如何遍历网络中的所有节点？BFS和DFS有什么本质区别？各自适用什么场景？**

## 直观思路：水波 vs 探险家

### BFS：水波扩散

想象在水池中心扔一块石头，水波逐层向外扩散。

**BFS（Breadth-First Search，广度优先搜索）**就像水波：
- 从起点开始，先访问所有"一步可达"的邻居
- 再访问所有"两步可达"的邻居
- 再访问所有"三步可达"的邻居
- 每层访问完，再向外扩散一层

**关键类比**：BFS找到的是**无权图的最短路径**。

因为BFS按"距离"逐层搜索，第一次找到目标节点时，走的步数就是最短的。

### DFS：探险家深入

想象一个探险家进入洞穴系统：
- 选一条路，深入探索
- 走到尽头，标记"已探索"，回头
- 换另一条路，继续深入
- 直到所有路径都标记完毕

**DFS（Depth-First Search，深度优先搜索）**就像探险家：
- 从起点开始，选一个邻居深入
- 沿这条路一直走到尽头
- 回头，换另一条邻居深入
- 直到所有可达节点都访问过

**关键类比**：DFS擅长**连通性检测**和**环检测**。

因为DFS深入探索，能找到所有可达节点（连通分量），也能发现"回头路"（环）。

### 关键区别

| 特性 | BFS | DFS |
|------|-----|-----|
| 搜索顺序 | 逐层扩散 | 深入后回头 |
| 数据结构 | 队列（先进先出） | 栈/递归（后进先出） |
| 找到的路径 | 无权最短路径 | 第一条可达路径 |
| 适用场景 | 最短路径、最短距离 | 连通性、环检测、拓扑排序 |
| 空间复杂度 | O(V)（队列） | O(V)（递归栈） |

## 规范定义：BFS和DFS的形式化

### BFS定义

**定义（BFS）**：从源点s开始，按"距离"顺序访问所有可达节点。

**BFS性质**：
1. **层次性**：节点按到s的距离分层，第k层节点距离为k
2. **最短路径**：第一次访问节点v时，dist[v]就是s到v的最短距离（无权）
3. **树结构**：BFS访问顺序形成BFS树，树边是最短路径上的边

**BFS伪代码**：
```
BFS(G, s):
    初始化：所有节点标记为未访问，距离为∞
    dist[s] = 0，将s加入队列
    
    while 队列不为空:
        u = 队列.pop()  # 取队首
        
        for v in u的邻居:
            if v未访问:
                dist[v] = dist[u] + 1
                parent[v] = u  # 记录路径
                将v加入队列
                标记v为已访问
    
    return dist, parent
```

### DFS定义

**定义（DFS）**：从源点s开始，尽可能深入探索，回溯后继续探索其他分支。

**DFS性质**：
1. **时间戳**：每个节点有两个时间戳
   - d[v]：发现时间（第一次访问）
   - f[v]：完成时间（所有邻居访问完）
2. **边分类**：DFS可以分类边为四种类型
   - 树边（Tree edge）：发现新节点的边
   - 后向边（Back edge）：指向祖先节点的边（**存在后向边⟺有环**）
   - 前向边（Forward edge）：指向已访问子孙的边
   - 交叉边（Cross edge）：指向其他分支的边
3. **树结构**：DFS访问顺序形成DFS树/森林

**DFS伪代码**：
```
DFS(G):
    time = 0
    初始化：所有节点标记为未访问
    
    for u in 所有节点:
        if u未访问:
            DFS_VISIT(u)
    
    return d, f, parent

DFS_VISIT(u):
    time++
    d[u] = time  # 记录发现时间
    标记u为灰色（正在探索）
    
    for v in u的邻居:
        if v白色（未访问）:
            parent[v] = u
            DFS_VISIT(v)  # 递归深入
        elif v灰色（正在探索）:
            发现后向边（u→v），说明有环
    
    time++
    f[u] = time  # 记录完成时间
    标记u为黑色（已完成）
```

### 颜色标记的意义

DFS用三种颜色标记节点状态：
- **白色（White）**：未访问
- **灰色（Gray）**：正在探索（已发现，邻居未全部访问）
- **黑色（Black）**：已完成（所有邻居都访问过）

**关键定理**：如果DFS过程中发现灰色节点v（正在探索），则存在后向边，说明图有环。

**证明**：灰色节点v是当前DFS路径上的节点（祖先），边u→v指向祖先，形成环。

## 算法实现：Python代码

### BFS实现

```python
from collections import deque

def bfs(G, s):
    """
    BFS：从s开始，返回到各点的距离和路径
    
    参数：
        G：邻接表表示的图，G[u] = [v1, v2, ...]（无权图的邻居列表）
        s：源点
    
    返回：
        dist：距离字典，dist[v]是s到v的最短距离
        parent：父节点字典，用于重建路径
    """
    dist = {}
    parent = {}
    
    # 初始化
    for v in range(len(G)):
        dist[v] = float('inf')
        parent[v] = None
    
    dist[s] = 0
    queue = deque([s])
    
    while queue:
        u = queue.popleft()
        
        for v in G[u]:
            if dist[v] == float('inf'):  # 未访问
                dist[v] = dist[u] + 1  # 无权图，每步距离为1
                parent[v] = u
                queue.append(v)
    
    return dist, parent

def reconstruct_path(parent, s, t):
    """从parent字典重建s到t的路径"""
    path = []
    v = t
    while v is not None and v != s:
        path.append(v)
        v = parent[v]
    if v == s:
        path.append(s)
        return path[::-1]  # 反转
    else:
        return None  # 无路径
```

### DFS实现

```python
def dfs(G):
    """
    DFS：遍历整个图，返回时间戳和边分类
    
    返回：
        d：发现时间字典
        f：完成时间字典
        parent：父节点字典
        edges：边分类（树边/后向边/前向边/交叉边）
    """
    time = 0
    d = {}
    f = {}
    parent = {}
    color = {}
    edge_types = []
    
    # 初始化
    for v in range(len(G)):
        color[v] = 'white'
        parent[v] = None
    
    def dfs_visit(u):
        nonlocal time
        time += 1
        d[u] = time
        color[u] = 'gray'  # 正在探索
        
        for v in G[u]:
            if color[v] == 'white':
                parent[v] = u
                edge_types.append((u, v, 'tree'))  # 树边
                dfs_visit(v)
            elif color[v] == 'gray':
                edge_types.append((u, v, 'back'))  # 后向边，有环！
            elif color[v] == 'black':
                if d[u] < d[v]:
                    edge_types.append((u, v, 'forward'))  # 前向边
                else:
                    edge_types.append((u, v, 'cross'))  # 交叉边
        
        time += 1
        f[u] = time
        color[u] = 'black'  # 已完成
    
    for u in range(len(G)):
        if color[u] == 'white':
            dfs_visit(u)
    
    return d, f, parent, edge_types

def has_cycle(edge_types):
    """根据边分类判断是否有环"""
    for u, v, edge_type in edge_types:
        if edge_type == 'back':
            return True
    return False
```

## 正确性分析：为什么BFS找最短路径

### BFS最短路径定理

**定理**：在无权图中，BFS第一次访问节点v时，dist[v]就是s到v的最短距离。

**证明（归纳法）**：

**基例**：dist[s] = 0，显然正确。

**归纳假设**：假设BFS正确计算了所有距离≤k的节点。

**归纳证明**：
- BFS访问节点按距离分层：第k层节点距离为k
- 节点v在第k+1层被访问时：
  - v是从某个第k层节点u访问的
  - dist[v] = dist[u] + 1 = k + 1
  - 如果存在更短路径（距离≤k），v应该在更早的层被访问
  - 因此dist[v] = k+1是最短距离

**结论**：BFS正确计算所有节点的最短距离。

### DFS环检测定理

**定理**：有向图G有环 ⟺ DFS过程中发现后向边。

**证明**：

**方向1（后向边⟹环）**：
- 后向边u→v指向灰色节点v
- 灰色节点v是DFS路径上的祖先
- 存在路径v→...→u（DFS树边）
- 边u→v闭合路径，形成环

**方向2（环⟹后向边）**：
- 设环为v₁→v₂→...→vₖ→v₁
- DFS访问v₁，沿环访问v₂→...→vₖ
- 当访问vₖ的邻居v₁时，v₁是灰色（DFS路径上）
- 边vₖ→v₁是后向边

**结论**：后向边存在 ⟺ 有环。

## 复杂度分析

### BFS复杂度

**时间复杂度**：O(V + E)

- 每个节点最多进入队列一次：O(V)
- 每条边最多检查一次：O(E)
- 总时间：O(V + E)

**空间复杂度**：O(V)

- 队列最多存储一层节点
- 在最坏情况下（如星状图），队列可能存储V-1个节点
- 空间：O(V)

### DFS复杂度

**时间复杂度**：O(V + E)

- 每个节点访问一次：O(V)
- 每条边检查一次：O(E)
- 总时间：O(V + E)

**空间复杂度**：O(V)

- 递归栈深度最多V（如链状图）
- 空间：O(V)

```python
def connected_components(G):
    """用DFS找到连通分量"""
    visited = set()
    components = []
    
    def dfs(u, component):
        visited.add(u)
        component.append(u)
        for v in G[u]:
            if v not in visited:
                dfs(v, component)
    
    for u in range(len(G)):
        if u not in visited:
            component = []
            dfs(u, component)
            components.append(component)
    
    return components
```

## 本节小结

### 核心问题回答

**问题**：如何遍历网络中的所有节点？BFS和DFS有什么本质区别？

**答案**：
- **BFS**：逐层扩散，适合找无权最短路径和最短距离
- **DFS**：深入探索，适合连通性检测、环检测、拓扑排序
- **本质区别**：数据结构（队列vs栈），搜索策略（层次vs深度）

### 关键洞察

1. **BFS找最短路径**：因为按距离逐层搜索，第一次找到就是最近
2. **DFS检测环**：后向边指向祖先节点，闭合路径形成环
3. **时间复杂度相同**：都是O(V + E)，但空间策略不同

### LLM时代视角

**LLM能执行BFS/DFS吗？**

MAGMA基准[MAGMA'24]测试了LLM执行图算法的能力，发现：
- LLM在显式图上执行BFS/DFS困难重重
- 需要算法指令增强和高级提示技术
- LLM擅长生成BFS/DFS代码，但难以正确执行多步迭代

**关键洞察**：LLM缺乏"工作记忆"来跟踪BFS队列或DFS栈的状态。
