## 直观思路：剥洋葱与等价类

### 拓扑排序：剥洋葱

想象洋葱的结构：外层→中层→内层。

**拓扑排序**就像剥洋葱：
- 先找"无依赖"的课程（洋葱外层）
- 选完这些，找下一批"无依赖"的课程（洋葱中层）
- 逐层剥开，直到所有课程选完

**关键类比**：
- 课程依赖关系 = 有向边（先修→后修）
- 选课顺序 = 拓扑排序（线性顺序）
- 循环依赖 = 环（拓扑排序失败）

### SCC：等价类

在社交网络中，有些群体"必互达"：
- Alice认识Bob，Bob认识Charlie，Charlie认识Alice
- 这个三人组内部互相可达，形成一个"强连通分量"

**SCC（Strongly Connected Component，强连通分量）**：
- 内部节点互相可达
- 是"必互达"的最大节点集
- SCC之间可能有单向依赖

**关键类比**：
- SCC = 等价类（必互达关系）
- SCC分解 = 把图划分为多个等价类
- SCC DAG = SCC之间的依赖关系（无环）

## 规范定义：DAG、拓扑排序、SCC

### DAG定义

**定义（DAG）**：有向无环图（Directed Acyclic Graph）是没有环的有向图。

**DAG性质**：
- 存在拓扑排序
- 至少有一个入度为0的节点（起点）
- 至少有一个出度为0的节点（终点）

### 拓扑排序定义

**定义（拓扑排序）**：对DAG的顶点进行线性排序，使得对于每条边u→v，u在v之前。

**拓扑排序存在性定理**：
- 有向图G有拓扑排序 ⟺ G是DAG（无环）

**证明**：
- 方向1（拓扑排序⟹无环）：如果拓扑排序存在，边u→v要求u在v前。如果存在环，环上节点无法排序（矛盾）。
- 方向2（无环⟹拓扑排序）：DAG至少有一个入度为0的节点。删除它后剩余图仍是DAG。递归删除所有入度为0节点，得到拓扑排序。

### SCC定义

**定义（SCC）**：强连通分量是节点集合C，满足：
- 内部可达性：C中任意两点u、v互相可达（u→v且v→u）
- 最大性：不存在更大的集合C'包含C且满足内部可达性

**SCC分解定理**：
- 任何有向图可以分解为若干SCC
- SCC之间形成DAG（无环）

**关键洞察**：SCC内部的环不影响拓扑排序（因为SCC内部互相可达），但SCC之间的依赖关系必须是DAG。

## 算法实现：Kahn拓扑排序和Kosaraju SCC

### Kahn拓扑排序

**Kahn算法思路**：
1. 计算所有节点的入度
2. 将入度为0的节点加入队列
3. 逐个删除入度为0的节点，更新剩余节点入度
4. 如果最终删除所有节点，拓扑排序成功；否则有环

```python
from collections import deque

def kahn_toposort(G):
    """
    Kahn拓扑排序算法
    
    参数：
        G：邻接表表示的有向图
    
    返回：
        order：拓扑排序结果（列表）
        或 None（如果存在环）
    """
    n = len(G)
    in_degree = [0] * n
    
    # 计算入度
    for u in range(n):
        for v, _ in G[u]:
            in_degree[v] += 1
    
    # 初始化队列（入度为0的节点）
    queue = deque([u for u in range(n) if in_degree[u] == 0])
    order = []
    
    while queue:
        u = queue.popleft()
        order.append(u)
        
        # 删除u，更新邻居入度
        for v, _ in G[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)
    
    # 检查是否所有节点都排序
    if len(order) == n:
        return order  # 拓扑排序成功
    else:
        return None  # 存在环
```

### DFS拓扑排序

**DFS拓扑排序思路**：
- DFS完成后，按完成时间逆序排列节点
- 完成时间晚的节点应该在拓扑排序的前面

```python
def dfs_toposort(G):
    """
    DFS拓扑排序算法
    
    思路：按完成时间逆序排列
    """
    n = len(G)
    color = ['white'] * n
    finish_order = []
    
    def dfs(u):
        color[u] = 'gray'
        for v, _ in G[u]:
            if color[v] == 'white':
                dfs(v)
        color[u] = 'black'
        finish_order.append(u)  # 记录完成顺序
    
    for u in range(n):
        if color[u] == 'white':
            dfs(u)
    
    # 逆序即为拓扑排序
    return finish_order[::-1]
```

### Kosaraju SCC算法

**Kosaraju算法思路**：
- 第一遍DFS：在原图上DFS，记录完成时间
- 第二遍DFS：在逆图上按完成时间逆序DFS，每次DFS访问的节点形成一个SCC

```python
def kosaraju_scc(G):
    """
    Kosaraju强连通分量算法
    
    思路：两遍DFS
    """
    n = len(G)
    visited = set()
    finish_order = []
    
    # 第一遍DFS：原图，记录完成顺序
    def dfs1(u):
        visited.add(u)
        for v, _ in G[u]:
            if v not in visited:
                dfs1(v)
        finish_order.append(u)
    
    for u in range(n):
        if u not in visited:
            dfs1(u)
    
    # 构建逆图
    G_rev = [[] for _ in range(n)]
    for u in range(n):
        for v, _ in G[u]:
            G_rev[v].append((u, 1))
    
    # 第二遍DFS：逆图，按逆完成顺序
    visited.clear()
    sccs = []
    
    def dfs2(u, scc):
        visited.add(u)
        scc.append(u)
        for v, _ in G_rev[u]:
            if v not in visited:
                dfs2(v, scc)
    
    for u in reversed(finish_order):
        if u not in visited:
            scc = []
            dfs2(u, scc)
            sccs.append(scc)
    
    return sccs
```

## 正确性分析

### Kahn算法正确性

**定理**：Kahn算法返回拓扑排序当且仅当图是DAG。

**证明**：
- 如果图是DAG，至少有一个入度为0的节点。删除后剩余图仍是DAG。递归删除所有入度为0节点，得到拓扑排序。
- 如果图有环，环上节点入度永远不为0，无法删除所有节点，返回None。

### Kosaraju算法正确性

**定理**：Kosaraju算法正确分解SCC。

**证明思路**：
- 第一遍DFS的完成时间：SCC内部节点完成时间相近
- 逆图中，SCC内部的边反向，但连通性不变
- 第二遍DFS按逆完成时间访问，保证每次DFS访问一个完整的SCC

|------|-----------|-----------|

## 本节小结

### 核心问题回答

**问题**：如何确定任务执行顺序？如何检测循环依赖？如何识别强连通分量？

**答案**：
- **拓扑排序**：确定任务顺序，要求图是DAG（无环）
- **环检测**：拓扑排序失败意味着有环
- **SCC分解**：识别"必互达"节点群，SCC之间形成DAG

### 关键洞察

1. **拓扑排序存在性**：拓扑排序⟺DAG（无环）
2. **Kahn算法直观**：剥洋葱，先删除无依赖节点
3. **SCC是等价类**：必互达关系划分图

### LLM时代视角

**DAG在LLM Agent中的应用**：

LLM Agent系统中的任务编排：
- 任务有依赖关系（如"先查询数据库，再生成报告"）
- 任务依赖图是DAG
- 拓扑排序确定执行顺序

**LLM能否执行拓扑排序？**

MAGMA基准显示LLM在拓扑排序上也有困难，但比BFS/DFS稍好。原因是拓扑排序的"剥洋葱"策略更直观。
