# 7.4 最小生成树——以最小成本连通网络

> **问题情境**：某电信公司要在10个城市间铺设光缆，成本与距离成正比。如何用最小总成本连通所有城市？

---

## 问题情境：光缆铺设的优化

小明在某电信公司实习，接到一个任务：规划10个城市的光缆铺设方案。

数据如下：
- 10个城市，每对城市间的铺设成本已知
- 目标：用最小总成本连通所有城市

小明的直觉：用最小成本连接所有城市，就是选成本最低的边。

但他遇到一个问题：
- 选最小的9条边，可能不连通（有些城市孤立）
- 选太多边，成本增加

他需要一种策略：**既保证连通，又最小化总成本**。

**这一节要回答的核心问题**：**如何用最小边权和连通所有节点？什么是"安全边"？贪心策略为什么正确？**

---

## 直观思路：城市扩张 vs 拼图组装

### Prim：从中心向外生长

想象城市从中心向外扩张：
- 从一个城市开始
- 找离它最近的城市，连接
- 找离已连通城市群最近的未连通城市，连接
- 重复直到所有城市连通

**Prim算法**：
- 从一个节点开始，形成"已连通集合"
- 每次选连接"已连通集合"和"未连通集合"的最小边
- 逐步扩张，直到所有节点连通

**关键类比**：Prim像城市扩张，从中心向外生长。

### Kruskal：从小边组装拼图

想象拼图组装：
- 先拼小块（最小的边）
- 小块连接成大块
- 大块连接成完整拼图

**Kruskal算法**：
- 把所有边按权重排序
- 从最小边开始，检查是否连接两个未连通的部分
- 如果连接，加入；否则跳过（会形成环）
- 重复直到所有节点连通

**关键类比**：Kruskal像拼图组装，从小块拼成完整图。

### 关键洞察

两种算法都是**贪心策略**：
- 每次选择当前最优（最小边）
- 为什么正确？**切割性质**保证

## 规范定义：生成树、切割性质、MST

### 生成树定义

**定义（生成树）**：对于连通无向图G = (V, E)，生成树T是：
- 子图：T的节点集合是V
- 树结构：T恰好有V-1条边，连通且无环

**性质**：
- 生成树连接所有节点
- 生成树是最小连通子图（去掉任何边就不连通）
- 生成树有V-1条边

### MST定义

**定义（MST）**：最小生成树是边权最小的生成树。

**形式化**：对于加权图G = (V, E, w)，MST T满足：
- T是生成树
- w(T) = Σ_{e∈T} w(e) 最小

### 切割性质

**定义（切割）**：把节点集合V划分为两部分S和V-S。

**切割性质定理**：对于任何切割(S, V-S)，跨越切割的**轻量级边**（权重最小的边）属于某棵MST。

**证明**：
- 设MST T不含轻量级边e（跨越切割）
- T必有一条边e'跨越切割（否则不连通）
- e比e'轻（权重小）
- 用e替换e'，得到T'，权重更小，矛盾
- 因此e必在某棵MST中

**关键洞察**：切割性质保证贪心选择的正确性。

## 算法实现：Prim和Kruskal

### Prim算法

**Prim算法思路**：
1. 从源点s开始，形成"已连通集合"S = {s}
2. 维护边集合：连接S和V-S的边
3. 选最小边e = (u, v)，u∈S，v∈V-S
4. 将v加入S，更新边集合
5. 重复直到S = V

```python
import heapq

def prim_mst(G, weights, start=0):
    """
    Prim算法：贪心 + 优先队列
    
    参数：
        G：邻接表表示的图
        weights：边权重字典 {(u, v): w}
        start：起始节点
    
    返回：
        mst：最小生成树的边列表 [(u, v, w)]
        total_weight：总权重
    """
    n = len(G)
    visited = set([start])
    mst = []
    
    # 初始化优先队列：从start出发的所有边
    pq = []
    for v, w in G[start]:
        heapq.heappush(pq, (w, start, v))
    
    while pq and len(visited) < n:
        w, u, v = heapq.heappop(pq)
        
        if v in visited:
            continue  # v已连通，跳过
        
        # 加入MST
        visited.add(v)
        mst.append((u, v, w))
        
        # 添加从v出发的新边
        for next_v, next_w in G[v]:
            if next_v not in visited:
                heapq.heappush(pq, (next_w, v, next_v))
    
    total_weight = sum(w for _, _, w in mst)
    return mst, total_weight
```

### Kruskal算法

**Kruskal算法思路**：
1. 把所有边按权重排序
2. 从最小边开始，检查是否连接两个未连通部分
3. 用并查集（Union-Find）维护连通性
4. 如果边(u, v)连接不同连通分量，加入MST
5. 重复直到MST有V-1条边

```python
class UnionFind:
    """并查集实现"""
    
    def __init__(self, n):
        self.parent = [i for i in range(n)]
        self.rank = [0] * n
    
    def find(self, x):
        """查找根节点（带路径压缩）"""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x, y):
        """合并两个集合（按秩合并）"""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False  # 已经在同一集合
        
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
        
        return True  # 成功合并

def kruskal_mst(G, weights):
    """
    Kruskal算法：贪心 + 并查集
    
    参数：
        G：邻接表表示的图
        weights：边权重字典 {(u, v): w}
    
    返回：
        mst：最小生成树的边列表 [(u, v, w)]
        total_weight：总权重
    """
    n = len(G)
    
    # 收集所有边并排序
    edges = []
    for u, v in weights.keys():
        w = weights[(u, v)]
        edges.append((w, u, v))
    edges.sort()
    
    uf = UnionFind(n)
    mst = []
    
    for w, u, v in edges:
        if uf.union(u, v):  # 连接不同连通分量
            mst.append((u, v, w))
            if len(mst) == n - 1:
                break  # MST完成
    
    total_weight = sum(w for _, _, w in mst)
    return mst, total_weight
```

## 正确性分析：切割性质证明贪心

### Prim正确性

**定理**：Prim算法返回MST。

**证明（基于切割性质）**：
- 每次Prim选的边e连接S和V-S，是跨越切割的轻量级边
- 切割性质保证e属于某棵MST
- 所有选的边都在MST中
- 最终选了V-1条边，形成生成树
- 因此Prim返回MST

### Kruskal正确性

**定理**：Kruskal算法返回MST。

**证明（基于切割性质）**：
- Kruskal每次选最小边e
- 如果e连接不同连通分量C₁和C₂，设切割S = C₁
- e是跨越切割(S, V-S)的轻量级边（因为已排序，e是最小的）
- 切割性质保证e属于某棵MST
- 所有选的边都在MST中
- 最终选了V-1条边，形成生成树
- 因此Kruskal返回MST

## 复杂度分析

### Prim复杂度

| 数据结构 | 时间复杂度 |
|---------|-----------|
| 数组（简单） | O(V²) |
| 二叉堆 | O(E log V) |
| 斐波那契堆 | O(E + V log V) |

**适用场景**：
- 稠密图：用数组，O(V²)更优
- 稀疏图：用二叉堆，O(E log V)更优

### Kruskal复杂度

| 操作 | 时间复杂度 |
|------|-----------|
| 边排序 | O(E log E) |
| 并查集操作 | O(E α(V))，α是反Ackermann函数（近似常数） |
| 总时间 | O(E log E) 或 O(E α(V)) |

**适用场景**：
- 稀疏图：Kruskal更优（边排序快）
- 边权重范围小：可以用计数排序优化到O(E)

### Prim vs Kruskal对比

| 特性 | Prim | Kruskal |
|------|------|--------|
| 数据结构 | 优先队列 | 并查集 |
| 稀疏图效率 | O(E log V) | O(E log E) ≈ O(E log V) |
| 稠密图效率 | O(V²)（数组）| O(E log E) ≈ O(V² log V) |
| 实现难度 | 中等 | 简单（需要并查集） |
| 边排序需求 | 不需要 | 需要 |

## 本节小结

### 核心问题回答

**问题**：如何用最小边权和连通所有节点？

**答案**：
- **Prim算法**：从一个节点扩张，每次选连接已连通和未连通集合的最小边
- **Kruskal算法**：从小边开始组装，每次选连接不同连通分量的最小边
- **正确性保证**：切割性质定理

### 关键洞察

1. **切割性质**：跨越切割的轻量级边必在某棵MST中
2. **贪心策略**：Prim和Kruskal都是贪心，切割性质保证正确
3. **两种策略**：Prim从中心扩张，Kruskal从小边组装

### LLM时代视角

**神经网络学习MST？**

MAGMA基准[MAGMA'24]显示LLM执行Prim算法困难。但研究显示：
- GNN可以学习近似MST
- 神经Steiner树求解：用深度学习解决NP-hard的Steiner树问题（比MST更难）

**关键洞察**：传统算法的迭代结构可以被神经网络"学习"。
