## 问题情境：供水系统的优化

小明在某供水公司实习，接到一个任务：计算从水源到用户的最大输水量。

系统结构：
```
    水源(S) → 管道1 → 中转站(A) → 管道3 → 用户(T)
              ↓         ↓
           管道2     管道4
              ↓         ↓
           中转站(B) → 管道5 → 用户(T)
```

管道容量：
- 管道1: 10单位/秒
- 管道2: 5单位/秒
- 管道3: 8单位/秒
- 管道4: 6单位/秒
- 管道5: 7单位/秒

小明需要计算：最大能输送多少水？

**这一节要回答的核心问题**：**如何最大化网络中的资源流量？什么是增广路径？最大流最小割定理是什么？**

---

## 直观思路：水流找路

### 流的基本概念

**流**：沿管道输送的水量。

**容量约束**：管道流量不超过容量。

**流量守恒**：中转站流入=流出（不囤水）。

**最大流**：从水源到用户的最大输送量。

### 直觉：找还能加水的路径

小明想：
- 当前送10单位水，管道1还能加吗？不行，已满。
- 检查其他管道，还有容量吗？
- 找一条"还能加水"的路径，增加流量
- 重复，直到找不到可加水的路径

**增广路径**：还能容纳更多流的路径。

**Ford-Fulkerson方法**：
- 找增广路径
- 沿路径增加流量
- 重复，直到找不到增广路径

---

## 规范定义：流网络、可行流、最大流最小割定理

### 流网络定义

**定义（流网络）**：G = (V, E, c, s, t)，其中：
- V：节点集合
- E：有向边集合
- c：容量函数，c(u, v)是边容量
- s：源点（source）
- t：汇点（sink）

### 可行流定义

**定义（可行流）**：流函数f满足：
1. **容量约束**：f(u, v) ≤ c(u, v)
2. **流量守恒**：对于中转节点v（v≠s且v≠t），流入=流出
   Σ_{u} f(u, v) = Σ_{w} f(v, w)

**流量值**：从源点发出的总流量，|f| = Σ_{v} f(s, v) - Σ_{u} f(u, s)

### 增广路径

**定义（增广路径）**：在残量图中，从s到t的路径。

**残量图**：边(u, v)的残量 = c(u, v) - f(u, v)，表示还能增加多少流。

**反向边**：如果f(u, v) > 0，存在反向边(v, u)，容量为f(u, v)，表示可以"撤回"流量。

### 最大流最小割定理

**定义（割）**：把V划分为S和T，s∈S，t∈T。

**割容量**：从S到T的边的容量之和。

**割流量**：从S到T的边的流量之和。

**定理（最大流最小割）**：最大流 = 最小割容量。

**证明思路**：
- 最大流 ≤ 最小割容量（流量不能超过割容量）
- 当找不到增广路径时，当前流 = 最小割容量
- 因此最大流 = 最小割容量

## 算法实现：Ford-Fulkerson和Edmonds-Karp

### Ford-Fulkerson方法

```python
from collections import deque

def ford_fulkerson(G, capacity, s, t):
    """
    Ford-Fulkerson方法：找增广路径，增加流量
    
    参数：
        G：邻接表表示的流网络
        capacity：容量字典 {(u, v): c}
        s：源点
        t：汇点
    
    返回：
        max_flow：最大流量值
        flow：流字典 {(u, v): f}
    """
    n = len(G)
    
    # 初始化流和残量
    flow = {e: 0 for e in capacity.keys()}
    # 正确初始化残量图：正向边=容量，反向边=0
    residual = {}
    for (u, v), c in capacity.items():
        if u not in residual:
            residual[u] = {}
        if v not in residual:
            residual[v] = {}
        residual[u][v] = c  # 正向边容量
        residual[v][u] = 0  # 反向边初始为0
    
    def bfs_find_path():
        """BFS找增广路径"""
        parent = {s: None}
        queue = deque([s])
        
        while queue:
            u = queue.popleft()
            for v in G[u]:
                if v not in parent and residual[u][v] > 0:
                    parent[v] = u
                    if v == t:
                        return parent
                    queue.append(v)
        return None
    
    max_flow = 0
    
    while True:
        parent = bfs_find_path()
        if not parent:
            break  # 找不到增广路径
        
        # 找最小残量（瓶颈）
        path_flow = float('inf')
        v = t
        while v != s:
            u = parent[v]
            path_flow = min(path_flow, residual[u][v])
            v = u
        
        # 更新流量和残量
        v = t
        while v != s:
            u = parent[v]
            flow[(u, v)] = flow.get((u, v), 0) + path_flow
            residual[u][v] -= path_flow
            residual[v][u] += path_flow  # 反向边
            v = u
        
        max_flow += path_flow
    
    return max_flow, flow
```

### Edmonds-Karp算法

**Edmonds-Karp**：Ford-Fulkerson + BFS找最短增广路径。

**时间复杂度**：O(V·E²)

**改进**：BFS保证增广路径最短，减少迭代次数。

---

## 正确性分析：最大流最小割定理

### 定理证明

**证明**：
1. **最大流 ≤ 最小割容量**：
   - 流必须通过割的边
   - 流量不超过割容量
   - 因此最大流 ≤ 最小割容量

2. **找不到增广路径 ⟹ 当前流 = 最小割容量**：
   - 当找不到增广路径，残量图中s和t不连通
   - 设S = s可达的节点，T = V - S
   - S到T的边残量为0（否则能增广）
   - 这些边的流量 = 容量（饱和）
   - 当前流 = S到T的流量 = 割容量

3. **结论**：最大流 = 最小割容量

---

## 复杂度分析

| 算法 | 时间复杂度 |
|------|-----------|
| Ford-Fulkerson | O(E·f_max)，f_max是最大流值 |
| Edmonds-Karp | O(V·E²) |
| Dinic | O(V²·E) 或 O(E·logV)（动态树） |

**关键洞察**：
- Ford-Fulkerson时间取决于最大流值，非整数容量可能不终止
- Edmonds-Karp用BFS，保证终止且效率更好

- 最小割 = S到V-S的边

## 本节小结

### 核心问题回答

**问题**：如何最大化网络中的资源流量？

**答案**：
- **Ford-Fulkerson方法**：找增广路径，增加流量，直到找不到
- **最大流最小割定理**：最大流 = 最小割容量
- **Edmonds-Karp**：BFS找最短增广路径，效率更好

### 关键洞察

1. **增广路径**：残量图中从s到t的路径，还能增加流量
2. **反向边**：允许"撤回"流量，保证正确性
3. **最大流最小割**：流量最大时，割容量最小
