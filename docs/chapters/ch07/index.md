# Ch7 图算法——网络的数学语言

> **核心问题**：如何高效地探索、连接、优化网络结构？

---

## 章节首页引言

### 网络无处不在

打开手机，你看到的是：
- 社交网络：朋友关系图谱
- 地图应用：道路网络导航
- 推荐系统：商品关联网络
- 知识图谱：概念关系网络

这些看似不同的应用，背后都是同一个数学对象——**图**。

图是网络的数学语言。当你问"从A到B怎么走最快"时，你在问最短路径；当你问"用最少的光缆连接所有城市"时，你在问最小生成树；当你问"如何安排课程顺序满足先修要求"时，你在问拓扑排序。

### 这一章要回答的问题

学完这一章，你将能够回答：

1. **表示问题**：如何在计算机中存储网络？邻接矩阵和邻接表各有什么优缺点？
2. **探索问题**：如何遍历网络中的所有节点？BFS和DFS有什么本质区别？
3. **结构问题**：如何识别网络中的"必互达"节点群？如何检测循环依赖？
4. **连接问题**：如何用最小成本连通整个网络？
5. **优化问题**：如何找到两点间的最优路径？
6. **分配问题**：如何最大化网络中的资源流量？
7. **建模问题**：如何将现实问题转化为图问题？

### 图算法的独特地位

图算法在整个算法体系中处于特殊位置：

- **承上**：综合运用数据结构（数组、链表、栈、队列、堆、并查集）
- **启下**：为高级主题铺路（贪心、动态规划、NP完备性）
- **桥梁**：连接算法与AI（图神经网络、知识图谱、Transformer注意力机制）

### LLM时代的新视角

图算法在LLM时代焕发新生：

| 经典问题 | LLM时代新问题 |
|---------|-------------|
| BFS遍历 | LLM能执行BFS吗？（MAGMA基准[MAGMA'24]：困难重重） |
| 最短路径 | 神经网络能学习最短路径吗？（神经Bellman-Ford） |
| 图表示 | 注意力矩阵是邻接矩阵吗？（Transformer = Graph） |
| 知识图谱 | LLM能构建和推理知识图谱吗？ |
| 图神经网络 | GNN与LLM如何协作？ |

**核心洞察**：LLM擅长算法实现，但建模决策——"这个问题应该建模为图吗？用什么表示？选什么算法？"——是人的价值所在。建模错误比算法错误代价更高。

### 三个为什么

1. **为什么图如此重要？**
   - 网络无处不在：社交、交通、推荐、知识
   - 图是网络的标准数学模型

2. **为什么需要多种图算法？**
   - 不同问题需要不同策略：探索、连接、优化、分配
   - 没有万能算法，只有适合问题的算法

3. **为什么建模比算法更重要？**
   - 建模决策决定问题是否可解
   - 错误的建模让最优算法解错误的问题
   - 建模错误比算法错误代价更高

### 前沿研究连接

- **MAGMA基准[MAGMA'24]**：系统评估LLM执行BFS/DFS/Dijkstra/Floyd-Warshall/Prim的能力，发现LLM在显式图推理上困难重重

- **Graph Transformers[GT'24]**：注意力机制 = 全连接图上的消息传递
- **知识图谱构建**：LLM从文本抽取实体和关系，构建结构化知识
