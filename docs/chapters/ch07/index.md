# Ch7 贪心算法——局部选择的全局效果

## 本章核心叙事

理解贪心的陷阱就是理解 LLM 的陷阱。LLM 的贪心解码（每步选最高概率 token）就是贪心算法——局部最优不保证全局最优。这章的核心逻辑是**警示性的**：贪心何时有效是一个深刻的问题，拟阵理论给出了数学充分条件，但更多问题需要 DP。

## 7.1 贪心策略的直觉与陷阱

贪心每步做局部最优选择。活动选择问题（最早完成优先）是贪心有效的经典案例——但只有无权版本如此。**加权活动选择需要 DP**（练习 16.1-5）。

## 7.2 贪心正确性的证明

贪心选择性质 + 最优子结构。交换论证法：假设最优解不含贪心选择，将最优解中的元素换成贪心选择——不更差。

## 7.3 拟阵理论——贪心正确性的数学充分条件

CLRS 独有：遗傳性 + 交换性 → GREEDY 返回最优。图拟阵 M_G 的 GREEDY 就是 Kruskal。加权任务调度也是拟阵实例。

**关键限制**：活动选择和 Huffman 不是拟阵实例，但贪心仍然正确。拟阵是充分条件而非必要条件。

## 7.4 Huffman 编码

最优前缀编码。Huffman 算法的正确性由两个引理证明：最低频率字符是某最优树中深度最大的兄弟（交换论证）；合并后归结为更小的子问题（最优子结构）。

## 7.5 分数背包 vs 0-1 背包

同一问题结构，不同可处理性。分数背包贪心有效，0-1 背包需要 DP。原因：0-1 选择创造重叠子问题，分数选择不创造。

## 7.6 拟阵之外——贪心何时失败

贪心在以下情况失败：(1) 缺乏最优子结构（最长简单路径）；(2) 贪心选择性质不成立（0-1背包）；(3) 子问题重叠需要 DP。

**LLM 时代映射**：贪心解码 = 每步选最高概率 token。贪心解码陷入重复、不探索替代路径——这正是贪心算法的同一陷阱。理解贪心算法何时有效，就是理解何时可以信任 LLM 的单步最优选择。

## 7.7 练习

**层次一（基础）**：
1. 用交换论证证明活动选择贪心的正确性
2. 实现 Huffman 编码，计算编码长度和压缩比
3. 给出贪心策略在 0-1 背包上失败的实例

**层次二（LLM 协同）**：
4. 让 LLM 对活动选择问题分别给出贪心解和 DP 解，对比结果
5. 让 LLM 在加权活动选择上用贪心策略，观察其失败——然后让 LLM 自己发现为什么贪心不够

**层次三（Agent 设计）**：
