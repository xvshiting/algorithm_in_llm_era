# 第8章 NP完全性与应对——验证比发现容易

## 本章核心叙事

NP 完全性不是一堵绝望的墙，而是**理解 LLM 能力边界的理论基础**。验证比发现容易——这恰好是 LLM 输出验证的哲学基础。近似算法的"储蓄账户+股票市场"组合策略直接映射 LLM 输出管道的设计。

## 8.1 归约——算法设计与复杂性理论的桥梁

归约是保持真值的翻译。双重含义：算法方向（B 有算法则 A 也有）和难性方向（A 有下界则 B 也有更紧的下界）。

四核心源问题选择：(1) 3-SAT（默认选择）、(2) 整数划分（涉大数）、(3) 顶点覆盖（图选择问题）、(4) 哈密顿路径（图排序问题）。3-SAT → 顶点覆盖的器件构造是形式化建模的范本（变量器件+子句器件）。

## 8.2 P vs NP——验证与发现的非对称性

类 P = 多项式时间可解。类 NP = 解可在多项式时间内验证。NP 不意味着"非多项式"。Cook 定理：SAT 是 NP 完全的——NP 中所有问题归约到 SAT。如果任何一个 NP 完全问题有多项式算法，则 P=NP。

**验证比发现容易 ↔ LLM 输出验证架构**：生成答案难、验证答案易。自我一致性解码、链式思维验证、辩论框架都基于此非对称性。

## 8.3 NP完全性证明体系

归约链：SAT → 3-SAT → 顶点覆盖 → 独立集 → 团 → TSP。六条实用归约原则：（1）源问题尽可能受限；（2）目标问题尽可能一般；（3）从四核心源选择；（4）放大不良选择的惩罚；（5）先想战略再构器件；（6）卡住时在算法搜索和难性证明间切换。

## 8.4 应对困难问题——近似、随机化与启发式

近似算法的"储蓄账户+股票市场"比喻：近似算法保证质量比例但可能不是最优，启发式期望更好但无保证。最好两者都跑，取较优者。

2-近似顶点覆盖（极大匹配）、3/2-近似 TSP（Christofides）、ln(n)-近似集合覆盖（贪心且紧致）。

模拟退火的 e^{ΔE/kT} 接受劣转移，代价函数必须给出部分信用。遗传算法——Skiena 明确不推荐。

## 8.5 练习

**层次一（基础）**：
1. 从 3-SAT 归约到顶点覆盖，画出器件构造
2. 证明近似比：2-近似顶点覆盖、ln(n)-近似集合覆盖
3. 比较 SA 的温度参数与 LLM 的采样温度——它们在数学上是同一个东西

**层次二（LLM 协同）**：
4. 让 LLM 证明一个简单的归约（如 CLIQUE → 独立集），评价其器件构造是否正确
5. 设计一个"近似算法 + LLM 启发式"的组合：先用近似算法得到保证质量的解，再用 LLM 尝试改进

**层次三（Agent 设计）**：
6. 设计归约识别器 Skill：输入问题描述，识别是否可能是 NP 完全的（对照四核心源问题），如果是，推荐近似算法
