## 三、NP完全性核心内容分析

### 3.1 P vs NP 的直觉引入

**核心叙事**（大纲已建立）:
> NP 完全性不是一堵绝望的墙，而是**理解 LLM 能力边界的理论基础**。
> 验证比发现容易——这恰好是 LLM 输出验证的哲学基础。

**教学引入建议**:
```
┌─────────────────────────────────────────────────────────────────────┐
│  验证 vs 发现的非对称性                                             │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  TSP 问题:                                                          │
│  ├── 发现最优路线: 指数时间（穷举所有可能）                        │
│  └── 验证给定路线最优: 多项式时间（计算总距离+比较）               │
│                                                                     │
│  LLM 对照:                                                          │
│  ├── 生成最优答案: 困难（指数搜索空间）                            │
│  └── 验证给定答案正确: 容易（Self-consistency, Debate）            │
│                                                                     │
│  教学要点:                                                          │
│  1. 从TSP引入验证/发现非对称性                                     │
│  2. 直接映射到LLM验证架构                                          │
│  3. 强调P≠NP是猜想而非定理                                         │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 归约的直观理解

**核心方法**（Z-028 已建立）:
> 归约是保持真值的翻译：将问题 A 的实例转换为问题 B 的实例，使 A(G) = B(Y)。

**经典器件构造展示**:

| 归约 | 器件构造 | 可视化建议 |
|------|----------|-----------|
| **SAT → 3-SAT** | 变量分组+辅助变量 | 文本替换演示 |
| **3-SAT → Vertex Cover** | 变量器件+子句器件 | 图形化 gadget |
| **Vertex Cover → CLIQUE** | 补图变换 | 图反转可视化 |
| **CLIQUE → TSP** | 城市构造+距离矩阵 | 距离矩阵设计 |

**3-SAT → Vertex Cover 器件详解**（Z-028）:
```
变量器件: v_i 和 v̄_i 间的边 → 选择一端 = 选择真值赋值
子句器件: 三角形中2/3顶点在覆盖中 → 未覆盖顶点连接到满足文字

总覆盖大小: n + 2c 当且仅当可满足
```

### 3.3 经典NP完全问题

| 问题 | 复杂性 | 典型归约来源 | LLM相关性 |
|------|--------|--------------|-----------|
| **SAT/3-SAT** | NP-complete | Cook定理根 | Prompt约束满足 |
| **Vertex Cover** | NP-complete | 3-SAT | 安全策略覆盖 |
| **CLIQUE** | NP-complete | Vertex Cover | 社交网络分析 |
| **Hamiltonian Path** | NP-complete | Vertex Cover | Agent路径规划 |
| **TSP** | NP-complete | Hamiltonian Path | 优化问题 |

## 四、LLM时代视角

### 4.1 LLM辅助理解归约

**应用场景**:
| 场景 | LLM角色 | 人类角色 |
|------|---------|----------|
| **器件构造** | 提供构造建议 | 验证正确性 |
| **等价性论证** | 生成证明草案 | 检查逻辑漏洞 |
| **反例构造** | 寻找边界条件 | 评估有效性 |

**LLM局限性**（Z-029）:
> 对NP完全问题，基于验证的设计（生成+验证）比假设完美生成更务实。

### 4.2 LLM验证NP完全性证明

**方法**:
```
┌─────────────────────────────────────────────────────────────────────┐
│  LLM验证归约流程                                                    │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  输入: 归约构造描述                                                 │
│                                                                     │
│  LLM检查:                                                           │
│  ├── 1. 变换是否多项式时间                                         │
│  ├── 2. gadget是否正确连接                                         │
│  ├── 3. 等价性论证是否完整                                         │
│  └── 4. 是否有遗漏边界条件                                         │
│                                                                     │
│  输出: 验证报告 + 改进建议                                          │
│                                                                     │
│  关键: LLM验证正确性（NP内），不生成新归约（NP困难）               │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 LLM在NP问题上的局限性

**核心问题**:
- 指数级搜索空间
- 无法保证找到最优解
- 验证可行但生成困难

**务实策略**（大纲已建立）:
> 近似算法 + LLM 启发式组合：先用近似算法得到保证质量的解，再用LLM尝试改进

## 五、教学设计思路

### 5.1 问题驱动引入方式

**建议引入顺序**:
```
1. TSP问题直观引入
   ├── "找最优路线难，验证最优路线易"
   └── 直接映射LLM验证架构

2. SAT问题
   ├── "能解SAT就能解所有NP问题"
   ├── Cook定理直觉解释

3. 归约思维
   ├── "把A翻译成B"
   ├── 器件构造可视化

4. 近似应对
   ├── "储蓄账户+股票市场"
   ├── 实务组合策略
```

### 5.2 从具体问题到抽象概念递进

| 阶段 | 问题 | 抽象概念 |
|------|------|----------|
| **阶段1** | TSP验证 vs 发现 | P vs NP直觉 |
| **阶段2** | SAT通用性 | Cook定理 |
| **阶段3** | 3-SAT → Vertex Cover | 归约方法论 |
| **阶段4** | 近似算法应对 | NP完全性不是绝望之墙 |

### 5.3 反例和边界条件展示

**关键反例**:
| 反例 | 教学价值 |
|------|----------|
| **最长简单路径** | 无最优子结构（历史依赖） |
| **国际象棋将杀** | NP-hard但不在NP中（验证需指数博弈树） |
| **最大团问题** | 不可常数因子近似（除非P=NP） |
| **贪心顶点覆盖** | Θ(log n)反而比简单2-近似更差 |

### 5.4 练习设计（灵活分层）

**基础练习**:
1. 从3-SAT归约到Vertex Cover，画出器件构造
2. 证明近似比：2-近似顶点覆盖、ln(n)-近似集合覆盖
3. 比较SA的温度参数与LLM的采样温度

**LLM协同练习**:
4. 让LLM证明简单归约（CLIQU→独立集），评价器件构造
5. 设计"近似算法+LLM启发式"组合
6. 用LLM验证归约等价性论证

**Agent设计练习**（可选）:
7. 设计归约识别器Skill
8. 设计NP完全性检测工具
9. 设计近似算法推荐器
