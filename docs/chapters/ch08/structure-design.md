### 8.3 NP完全性与Cook定理——问题的等价性

**问题情境**
> 1971年，Stephen Cook证明SAT是"NP中最难的问题"。为什么？

**直觉建立**
Cook定理直觉：任何NP问题都可转化为SAT，SAT是"通用语言"

**规范定义**
- NP-hard定义：所有NP问题都可归约到它

**直觉建立的三步过程**

第一步：感受时间差距
- 发现问题：指数级搜索空间
- 验证问题：多项式级计算
- 差距：从"宇宙年龄"到"瞬间"

第二步：理解非对称性
- 发现：需要探索所有可能
- 验证：只需检查给定答案
- 关键：给定答案大大降低了难度

第三步：联系LLM
- LLM生成答案 = 发现问题（困难）
- LLM验证答案 = 验证问题（容易）
- 启示：设计"生成+验证"架构

**算法实现详细**
```python
def tsp_verify(cities, distances, route, claimed_optimal):
    """
    TSP验证器：判断给定路线是否最优（假设已知最优值）
    
    参数：
    - cities: 城市列表
    - distances: 距离矩阵 distances[i][j] 表示城市i到j的距离
    - route: 待验证的路线（城市索引列表）
    - claimed_optimal: 声称的最优距离
    
    返回：True 如果路线总距离等于声称最优值，False 否则
    """
    # 验证路线是否经过所有城市恰好一次
    if set(route) != set(range(len(cities))):
        return False
    
    # 计算总距离
    total = 0
    for i in range(len(route) - 1):
        total += distances[route[i]][route[i+1]]
    total += distances[route[-1]][route[0]]  # 回到起点
    
    # 验证是否等于声称最优值
    return abs(total - claimed_optimal) < 1e-9

# 复杂度分析
# 时间复杂度：O(n) - 遍历路线一次
# 空间复杂度：O(1) - 只存储累加和
```

**Cook定理直觉理解**
```
┌─────────────────────────────────────────────────────────────────────┐
│  Cook定理：为什么SAT是NP完全的？                                    │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  定理陈述（Cook, 1971）：                                           │
│  SAT问题是NP完全的。                                                │
│                                                                     │
│  证明直觉：                                                         │
│  任何NP问题都可以在多项式时间内转换为SAT实例                        │
│                                                                     │
│  ────────────────────────────────────────────────────────────────  │
│                                                                     │
│  第一步：NP问题的本质                                               │
│  ├── NP问题 = 存在多项式时间验证器V(x, c)                          │
│  ├── x：问题实例                                                   │
│  ├── c：证书（解）                                                 │
│  └── V在多项式时间内验证c是否是x的有效解                           │
│                                                                     │
│  第二步：验证器 = 图灵机程序                                        │
│  ├── 验证器是一个确定性图灵机                                      │
│  ├── 运行时间是多项式p(|x|)                                        │
│  ├── 状态转移、读写带都有确定规则                                  │
│  └── 这些规则可以编码为布尔约束                                    │
│                                                                     │
│  第三步：图灵机 → SAT                                               │
│  ├── 变量：表示图灵机在每个时刻的状态                              │
│  ├── 子句：确保状态转移正确                                        │
│  ├── 可满足：图灵机接受                                            │
│  └── 不可满足：图灵机拒绝                                          │
│                                                                     │
│  ────────────────────────────────────────────────────────────────  │
│                                                                     │
│  为什么SAT是"通用语言"？                                           │
│  ├── 任何计算都可以表达为布尔约束                                  │
│  ├── SAT可以模拟任何图灵机                                         │
│  └── 解SAT = 解任何NP问题                                          │
│                                                                     │
│  Cook的贡献：                                                       │
│  将非确定性图灵机的计算过程系统地编码为SAT公式                      │
│  这是NP完全性理论的基石                                            │
└─────────────────────────────────────────────────────────────────────┘
```

### 8.4 归约——问题间的翻译艺术

**Skiena六法则详解**
```
┌─────────────────────────────────────────────────────────────────────┐
│  Skiena归约六法则                                                   │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  【法则1】源问题尽可能受限                                          │
│  ─────────────────────────────────────────────                      │
│  原因：受限版本更简单，更容易构造器件                              │
│  例如：                                                            │
│  ├── 用3-SAT而非SAT（每子句恰好3文字）                            │
│  ├── 用3-顶点覆盖而非顶点覆盖（每个顶点最多连接3条边？）          │
│  └── 受限减少了需要考虑的情况                                      │
│                                                                     │
│  【法则2】目标问题尽可能一般                                        │
│  ─────────────────────────────────────────────                      │
│  原因：一般版本有更多自由度，更容易归约到                          │
│  例如：                                                            │
│  ├── 归约到Vertex Cover（而非3-Vertex Cover）                     │
│  ├── 增加自由度让器件设计更灵活                                    │
│  └── 不要过度约束目标问题                                          │
│                                                                     │
│  【法则3】从四核心源选择                                            │
│  ─────────────────────────────────────────────                      │
│  四核心源问题：                                                     │
│  ├── 3-SAT（布尔约束）                                             │
│  │   适用：逻辑、选择、约束满足问题                               │
│  ├── 整数划分（数值约束）                                          │
│  │   适用：分配、分割、数值问题                                   │
│  ├── 顶点覆盖（图约束）                                            │
│  │   适用：图问题、覆盖、选择问题                                 │
│  └── 哈密顿路径（路径约束）                                        │
│      适用：路径、环、序列问题                                      │
│                                                                     │
│  【法则4】大胆放大不良选择的惩罚                                    │
│  ─────────────────────────────────────────────                      │
│  原因：确保只有正确选择才可行                                      │
│  例如：                                                            │
│  ├── 3-SAT → VC：变量器件确保只能选x或¬x之一                      │
│  ├── 用大权重、特殊结构强制约束                                    │
│  └── 让错误选择导致目标问题无解                                    │
│                                                                     │
│  【法则5】先想战略再构小器件                                        │
│  ─────────────────────────────────────────────                      │
│  步骤：                                                            │
│  1. 理解两个问题的核心结构                                         │
│  2. 设计整体布局（战略）                                           │
│  3. 构造关键器件（战术）                                           │
│  4. 连接器件确保正确性                                             │
│                                                                     │
│  避免：一开始就陷入器件细节                                         │
│                                                                     │
│  【法则6】卡住时在算法搜索和难性证明间切换                         │
│  ─────────────────────────────────────────────                      │
│  原因：两种视角互相启发                                            │
│  ├── 找算法失败 → 发现问题的难度结构                              │
│  ├── 证明难性失败 → 发现问题的特殊结构                            │
│  └── 切换视角有助于突破思维瓶颈                                    │
└─────────────────────────────────────────────────────────────────────┘
```

**归约构造思维过程**
```
┌─────────────────────────────────────────────────────────────────────┐
│  归约构造：从战略到器件（3-SAT → Vertex Cover示例）                │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  【理解问题核心】                                                   │
│  3-SAT的核心：                                                     │
│  ├── 每个变量选真或假                                              │
│  └── 每个子句至少一个文字为真                                      │
│                                                                     │
│  Vertex Cover的核心：                                              │
│  ├── 选择一些顶点                                                  │
│  └── 每条边至少一个端点被选中                                      │
│                                                                     │
│  【战略设计】                                                       │
│  如何对应？                                                        │
│  ├── 变量选择 → 顶点选择                                           │
│  │   变量x和¬x只能选一个 → 选x或选¬x对应的顶点                    │
│  ├── 子句满足 → 边覆盖                                             │
│  │   子句至少一个文字真 → 至少一个文字对应的顶点被选中            │
│  └── 覆盖大小限制：n + 2c                                          │
│                                                                     │
│  【器件构造】                                                       │
│  变量器件：                                                         │
│  ├── 对每个变量x，创建两个顶点：x和¬x                             │
│  ├── 在x和¬x之间连一条边                                           │
│  ├── 覆盖这条边必须选x或¬x之一                                    │
│  └── = 变量必须赋真或假                                            │
│                                                                     │
│  子句器件：                                                         │
│  ├── 对每个子句(x ∨ y ∨ z)，创建一个三角形                        │
│  ├── 三角形的三个顶点对应三个文字                                  │
│  ├── 覆盖三角形需要选2个顶点                                       │
│  └── 第3个顶点（未选）必须通过连接到满足的文字来覆盖              │
│                                                                     │
│  连接设计：                                                         │
│  ├── 子句器件中未覆盖的顶点连接到满足的文字                       │
│  ├── 如果文字满足，对应的顶点被选中                                │
│  └── 这样第3个顶点被覆盖                                           │
│                                                                     │
│  【等价性论证】                                                     │
│  正向：3-SAT可满足 → 存在n+2c覆盖                                  │
│  ├── n个变量器件：每个选1个顶点（对应变量真值）                   │
│  ├── 2c个子句器件顶点：每个子句选2个                               │
│  └── 第3个顶点连接到满足的文字，已被选中                          │
│                                                                     │
│  反向：存在n+2c覆盖 → 3-SAT可满足                                  │
│  ├── 每个变量器件恰选1个顶点（因为要覆盖那条边）                   │
│  ├── 这给出了变量真值赋值                                          │
│  └── 每个子句至少一个文字真（否则无法覆盖）                       │
└─────────────────────────────────────────────────────────────────────┘
```

```
│                                                                     │

│    (x1 ∨ x2 ∨ z) ∧ (x1 ∨ x2 ∨ ¬z)                                 │

│                                                                     │

│        ○ ──┐                                                       │
```

### 8.7 应对NP完全问题——近似与启发式

**近似算法详解**
```
┌─────────────────────────────────────────────────────────────────────┐
│  顶点覆盖2-近似算法详解                                             │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  算法：                                                             │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Approx-Vertex-Cover(G)                                      │   │
│  │  1. C ← ∅                                                   │   │
│  │  2. E' ← E(G)                                               │   │
│  │  3. while E' ≠ ∅ do                                         │   │
│  │  4.     任选一条边(u, v) ∈ E'                               │   │
│  │  5.     C ← C ∪ {u, v}                                      │   │
│  │  6.     从E'中删除所有与u或v相连的边                        │   │
│  │  7. return C                                                 │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  示例演示：                                                         │
│  图：1──2──3──4                                                    │
│                                                                     │
│  第1轮：选边(2,3)，加入C={2,3}，删除边(1,2),(2,3),(3,4)            │
│  E'为空，返回C={2,3}                                               │
│                                                                     │
│  最优解：{2,3}，近似解：{2,3}，正好最优！                          │
│                                                                     │
│  另一个例子：                                                       │
│  图：    1                                                          │
│         /│\                                                         │
│        2─3─4                                                        │
│                                                                     │
│  第1轮：选边(1,2)，加入C={1,2}，删除边(1,2),(1,3),(1,4)            │
│  第2轮：选边(2,3)，加入C={1,2,2,3}，删除边(2,3),(3,4)              │
│  E'为空，返回C={1,2,3}                                             │
│                                                                     │
│  最优解：{1}，近似解：{1,2,3}，近似比=3                            │
│                                                                     │
│  近似比证明：                                                       │
│  ├── 设算法选择了k条边，则|C| = 2k                                 │
│  ├── 最优解OPT必须覆盖这k条边                                      │
│  ├── 每个顶点最多覆盖一条被选边（因为边不共享顶点）                │
│  ├── 因此OPT ≥ k                                                   │
│  └── |C|/OPT = 2k/k = 2，即近似比为2                               │
└─────────────────────────────────────────────────────────────────────┘
```

**启发式方法比较**
```
┌─────────────────────────────────────────────────────────────────────┐
│  启发式方法比较表                                                    │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                     │
│  方法         | 优点              | 缺点              | 适用场景    │
│  ────────────┼──────────────────┼──────────────────┼────────────  │
│  局部搜索     | 简单易实现        | 易陷入局部最优    | 中小规模    │
│              | 快速              | 对初始解敏感      │              │
│  ────────────┼──────────────────┼──────────────────┼────────────  │
│  模拟退火     | 能跳出局部最优    | 参数调优复杂      │ 大规模      │
│              | 有理论支撑        | 速度较慢          │ 组合优化    │
│  ────────────┼──────────────────┼──────────────────┼────────────  │
│  遗传算法     | 全局搜索能力强    | 参数多            │ 复杂问题    │
│              | 并行化容易        | 可能早熟收敛      │ 多目标      │
```

## 综合练习设计

### 基础练习

1. **问题情境与直觉**
   - 举一个日常生活中的"验证易、发现难"的例子（非TSP），并解释其与NP理论的关联

2. **方法应用与实现**
   - 实现SAT问题的验证器：给定布尔公式和变量赋值，判断公式是否可满足
   - 设计SAT到3-SAT的归约算法，并分析时间复杂度

3. **错误诊断与反例**
   - 诊断以下归约构造的错误："把每个变量变成一个顶点，每个子句变成一条边"
   - 构造一个反例：说明贪心选度最大顶点的方法不能保证常数近似比

### 进阶练习

4. **方案比较与权衡**
   - 比较顶点覆盖问题的三种策略：
     - 精确算法（分支限界）：小规模
     - 2-近似算法：有质量保证
     - 启发式（贪心选度最大）：实际效果好
   - 分析各自适用场景

5. **变式与边界条件**
   - 如果图是树结构，顶点覆盖问题是否多项式时间可解？为什么？
   - 如果限制每个顶点度数不超过3，问题复杂度如何变化？

### 开放设计

6. **综合应用**
   - 选择一个实际问题（如课程排课、物流配送），判断其是否NP完全
   - 如果是，设计一个近似算法或启发式方法
   - 分析算法的正确性和复杂度

7. **Agent设计练习**
   - 设计归约识别器Skill的接口和流程
   - 要求：输入问题描述，输出是否NP完全的判断及归约建议

### LLM协同实验

8. **验证归约正确性**
   - 让LLM验证3-SAT到Vertex Cover归约的正确性
   - 评价LLM的论证是否完整，是否有遗漏

9. **启发式设计**
   - 让LLM设计一个TSP的启发式规则
   - 在小规模实例上测试效果，评价其优劣

10. **理论探讨**
    - 与LLM讨论：如果P=NP，密码学会发生什么变化？
    - 评价LLM的回答是否正确，是否有遗漏
