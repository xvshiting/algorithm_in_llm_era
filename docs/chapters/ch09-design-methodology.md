# 第9章 算法设计方法论——设计算法的算法

## 本章核心叙事

这不是"传统方法论 + LLM 应用"，而是**方法论本身就是一个算法**——一个设计算法的算法。六问诊断法直接翻译为 Agent 的规划循环。把这六问编码为 Skill 就是终章的 Agent 设计练习。

## 9.1 六问诊断法

1. **我真的理解问题吗？** — 定义输入/输出，构造小例子，评估规模/速度约束
2. **简单算法或启发式？** — 暴力、贪心规则；分析正确性和失败模式
3. **我的问题在目录中吗？** — 查已有知识和实现
4. **特殊情况可解？** — 删减参数、设为平凡值、先简化再推广
5. **哪种范式适合？** — 排序、分治、DP、数据结构、随机化/SA、LP/IP、NP完全性
6. **还是想不出？** — 请教专家，回到问题1

正确回答"能不能这样做"不是"不能"而是"不能，因为……"。

## 9.2 从六问到 Agent 规划循环

六问法每个步骤都可以编码为 Agent 的决策节点：

| 六问 | Agent 对应 |
|------|-----------|
| 1. 理解问题 | 问题解析 + 输入输出规范 |
| 2. 简单算法 | 基线生成 + 正确性自检 |
| 3. 查目录 | 知识库检索（RAG） |
| 4. 特殊情况 | 问题简化 + 逐步推广 |
| 5. 范式匹配 | 分类器 + 模板选择 |
| 6. 循环 | 重新审视 + 专家调用 |

## 9.3 试飞员心态

Skiena 的比喻：即将坠毁的试飞员不恐慌——他们跑过可能行动的清单。太多人面对设计问题会冻结。遵循问题序列，大多数情况下你会知道下一步做什么。

**人机类比的映射**：LLM 不恐慌但会"冻结"——生成自信但错误的输出。试飞员心态（系统地跑清单而非跳到结论）正是 chain-of-thought with self-correction 试图实现的。

## 9.4 从简单到正确，从正确到最优

先给一个简单正确但慢的算法，然后再优化。面试官想看的是思维过程而非最终答案。这个原则贯穿全书：先暴力解，再分治优化；先贪心，再 DP 保证最优。

## 9.5 策略先于战术

"战略是寻求全局画面——构建通向目标的框架。战术是沿途的小战役。没有全局战略，战术毫无意义。"——Skiena。战略问题："这能建模为图算法问题吗？"战术问题："用邻接表还是邻接矩阵？"

## 9.6 练习

**层次一（基础）**：
1. 用六问法分析一个你不熟悉的问题（如集合覆盖），记录每步的推理
2. 对一个 NP 完全问题，分别给出暴力解、贪心启发式和近似算法

**层次二（LLM 协同）**：
3. 让 LLM 对同一问题走六问法，你评价每步的质量——哪步 LLM 表现好？哪步需要人介入？
4. 构造一个"简单正确但慢"的 LLM 解和一个"复杂但快"的人写解，对比

**层次三（Agent 设计）**：
5. **终章项目**：把六问法编码为一个完整的 Agent Skill——输入问题描述，自动执行六问诊断，输出推荐范式和实现框架。这是本课程 Agent 设计的最高阶练习
