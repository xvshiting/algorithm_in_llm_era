# 第2章 练习

## 证明

1. 证明 BUILD-MAX-HEAP 的线性时间复杂度（高度分布 ⌈n/2^{h+1}⌉ → 总代价 O(n)）
2. 用势能法分析动态表扩张的摊还代价——解释为什么"半满时减半"导致抖动，而"1/4时减半"不抖动

## 实现

3. 实现红黑树的插入和删除（含双重黑色处理）
4. 实现并查集（按秩合并 + 路径压缩），测试 O(mα(n)) 的实际性能

## 建模

5. 给定三个应用场景，选择合适的数据结构并分析摊还复杂度：
   - 搜索引擎倒排索引（高频插入+范围查询）
   - 实时任务调度（优先级队列）
   - LRU 缓存（O(1) 查找+删除）

**1.1** 总代价 = Σ_{h=0}^{⌊lg n⌋} ⌈n/2^{h+1}⌉ · O(h) = O(n · Σ h/2^h) = O(n · 2) = O(n)

**2** 势函数 Φ(T)：α ≥ 1/2 时 Φ = 2·num-size，α < 1/2 时 Φ = size/2-num。朴素策略在 α=0.5 时反复触发扩缩容 → O(n) 摊还，正确策略 α < 1/4 减半 → O(1) 摊还。
