# 第2章 练习

## 层次一：基础

### 1.1 建堆线性时间

证明 BUILD-MAX-HEAP 的线性时间复杂度：
- 证明高度 h 的节点数 ≤ ⌈n/2^{h+1}⌉
- 利用高度分布推导总代价 O(n)

### 1.2 红黑树插入

给出红黑树插入的三种情况（情况1/2/3），画出旋转和重着色过程。解释"双重黑色"概念如何帮助处理删除。

### 1.3 摊还分析

用势能法分析动态表扩张的摊还代价：
- 定义势函数 Φ
- 证明插入操作的摊还代价为 O(1)
- 解释为什么朴素的"半满时减半"策略会导致抖动

### 1.4 散列表

比较链式散列、开放寻址、全域散列的优劣势。计算在负载因子 α = n/m 下各方法的期望查找时间。

## 层次二：LLM 协同

### 2.1 循环不变式审查

让 LLM 实现建堆操作 MAX-HEAPIFY，验证其循环不变式是否正确声明和证明。

### 2.2 数据结构选择

给定三个应用场景，让 LLM 推荐数据结构并分析摊还复杂度：
- 搜索引擎倒排索引（高频插入+范围查询）
- 实时任务调度（优先级队列）
- LRU 缓存（O(1) 查找+删除）

评价 LLM 的推荐是否考虑了摊还代价。

## 层次三：Agent 设计

### 3.1 数据结构推荐 Skill

设计一个 Skill：输入应用场景（操作类型、规模、延迟要求），自动推荐数据结构并生成实现框架。

- 场景特征：插入频率、查找频率、删除频率、是否需要有序遍历
- 输出：推荐数据结构 + 摊还复杂度分析 + 伪代码框架
- 这是数据结构扩张方法论（定理14.1）的应用

<summary>参考答案提示</summary>

**1.1** 高度 h 节点数 ≤ ⌈n/2^{h+1}⌉。总代价 = Σ_{h=0}^{⌊lg n⌋} ⌈n/2^{h+1}⌉ · O(h) = O(n · Σ h/2^h) = O(n · 2) = O(n)。

**1.3** 势函数 Φ(T) = 2·num[T] - size[T]（α ≥ 1/2 时），Φ(T) = size[T]/2 - num[T]（α < 1/2 时）。插入摊还代价 = 实际代价 + ΔΦ = O(1) + O(1) = O(1)。朴素策略在 α = 0.5 时反复插入删除导致 O(n) 搬移。
