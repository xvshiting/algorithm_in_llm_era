# 第3章 练习

## 层次一：基础

### 1.1 决策树下界

证明排序的 Ω(n lg n) 下界：
- n! 个排列 ≤ 2^h 片叶 → h ≥ lg(n!) = Ω(n lg n)
- 用信息论论证：每次比较最多 1 bit 信息，排序 n! 种排列至少需要 lg(n!) bit

### 1.2 快排的正确性

用循环不变式证明 PARTITION 的正确性。构造使确定性快排退化为 Θ(n²) 的输入（McIlroy 杀手序列的基本思想）。

### 1.3 排序三元悖

解释"O(n) 时间 + 稳定 + 原地"三元悖：三者只能同时取二。给出每种取二的例子。

### 1.4 线性时间选择

BFPRT 算法为什么分组大小选 5 而不是 3 或 7？计算不同分组大小的递推式和复杂度。

### 1.5 哈希概率分析

设散列表大小 m，n 个键用链式散列。证明：④ 链式散列期望查找时间 Θ(1 + α)；② 全域散列碰撞概率 ≤ 1/m。

## 层次二：LLM 协同

### 2.1 构造杀手序列

让 LLM 实现随机化快排，然后构造使其退化为 Θ(n²) 的输入。讨论：为什么随机化快排消除了对输入分布的依赖？

### 2.2 哈希策略对比

给定一个应用场景（如搜索引擎去重、拼写检查、缓存），比较三种哈希策略（链式、开放寻址、Bloom 过滤器）的优劣。让 LLM 分析，你评判。

### 2.3 排序三元悖与 LLM 推理

排序三元悖在 LLM 推理中的体现：
- 自回归生成：稳定（确定性顺序）但不是 O(1)
- beam search：更"原地"但可能不稳定
- 讨论这一类比的有效性和局限性

## 层次三：Agent 设计

### 3.1 排序策略自动选择 Skill

设计一个 Skill：输入数据特征（规模、分布、键类型、是否需要稳定），自动在确定性排序、随机化排序和哈希之间选择策略。

要求：
- 小规模 / 几乎有序 → 插入排序
- 大规模通用 → 随机化快排
- 键范围有限 → 计数/基数排序
- 需要稳定 → 归并排序
- 只需存在性检查 → Bloom 过滤器
