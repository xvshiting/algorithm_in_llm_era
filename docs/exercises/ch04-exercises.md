## 层次一：基础

### 1.1 DFS 与环检测

证明：有向图中 DFS 发现后向边当且仅当图有环（白色/灰色/黑色路径定理）。

### 1.2 MST 切割性质

用切割性质证明 Kruskal 算法的正确性：
- 设割 (S, V-S)，轻量级边 (u,v) 必在某棵 MST 中
- Kruskal 每步选择当前最小权重的安全边

## 层次二：LLM 协同

### 2.1 图建模评价

给出一个现实问题（如课程排课、网络路由），让 LLM 建模为图问题。评价其建模是否正确——建模错误比算法错误更致命。

判断标准：
- 顶点和边的定义是否正确？
- 权重/方向的设定是否合理？
- 是否遗漏了重要约束？

### 2.2 构造环检测遗漏

用 LLM 实现拓扑排序，构造让 LLM 忽略环检测的有向图测试用例。讨论：为什么 LLM 容易忽略"不可能"的情况？

## 层次三：Agent 设计

### 3.1 图问题识别与建模 Agent

设计 Agent：输入自然语言问题描述，自动执行：
1. 判断是否为图问题（关键词识别 + 语义分析）
2. 建模为图问题（定义顶点、边、权重、方向）
3. 选择标准图算法
4. 生成实现

核心难点：建模而非执行。Agent 的图建模能力是其天花板。

**1.1** ⇒：如果 DFS 发现后向边 (u,v)，则存在 v→...→u→v 的环。⇐：如果有环 C，设 C 中第一个发现的顶点为 v，则 C 中 v 的前驱 u 在发现 v 时是灰色的，(u,v) 是后向边。

**1.2** 切割性质：设 (S, V-S) 是尊重某 MST 的割，(u,v) 是 crossing 的轻量级边，则 (u,v) 安全。Kruskal 在每步选择的边跨越某个割且是轻量级的，因此是安全边。
