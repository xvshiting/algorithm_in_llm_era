# 第6章 练习

## 层次一：基础

### 1.1 剪贴证明

用剪贴证明论证矩阵链乘法的最优子结构：
- 假设 A₁...Aₙ 的最优括号化在 A_k 处分裂
- 如果子链 A₁...A_k 的括号化不是最优的，替换为最优的 → 总代价更小 → 矛盾

### 1.2 编辑距离统一框架

实现编辑距离的统一框架（一个 DP 表 + 四个桩函数）：
- 替换代价：c_sub = cost(a_i, b_j)
- 插入代价：c_ins = cost(-, b_j)
- 删除代价：c_del = cost(a_i, -)
- 匹配代价：c_match = 0（或某正数）

通过修改桩函数，分别实现 LCS、子串匹配、最大单调子序列。

### 1.3 最优子结构的反例

证明最长简单路径缺乏最优子结构：
- 构造一个图，使 u→v 的最长简单路径的子路径不是对应子问题 u→w 的最长简单路径
- 解释为什么路径历史使其不满足最优子结构

### 1.4 子问题图

画出矩阵链乘法 n=4 的子问题图（DAG），标注子问题数量和依赖边数。验证运行时间 O(|V| + |E|)。

## 层次二：LLM 协同

### 2.1 DP 递推式验证

让 LLM 为一个新问题（如书本分页、最优 BST）给出 DP 递推式。你用最优子结构的剪贴证明验证：
- 子问题是否有重叠？
- 最优子结构是否成立？
- 参数空间是否多项式？

### 2.2 贪心 vs DP 对比

让 LLM 对同一问题（如书本分页）分别给出贪心解和 DP 解：
- 贪心解可能"很好"但非最优
- DP 始终最优但有更高时间复杂度
- 讨论何时贪心足够好

## 层次三：Agent 设计

### 3.1 DP 识别器 Skill

设计 DP 识别器 Skill：输入问题描述，自动判断是否适合 DP：

检查清单：
1. 是否有重叠子问题？（暴力递归会重复计算同样的子问题吗？）
2. 是否有最优子结构？（子问题的最优解能否组合成全局最优？）
3. 参数空间是否多项式？（状态数是否 O(n^k)？）

如果三个条件都满足 → 推荐 DP，输出递推式和实现框架。
如果不满足 → 推荐贪心/回溯/近似算法，说明原因。

**1.3** 反例：图 a-b-c-d，边权：ab=2, bc=2, cd=2, ad=1。a→d 的最长简单路径 a-b-c-d 长度6，但子路径 a→c 的最长简单路径是 a-b-c 长度4，而 a-d-c 长度3 < 4。子路径 a→c 和后半路径 c→d 共享顶点，拼接后 a-b-c-d 不是简单路径（包含了 c-d 和 a-d-c 中的 d）。
