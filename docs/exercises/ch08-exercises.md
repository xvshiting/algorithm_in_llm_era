# 第8章 练习

## 层次一：基础

### 1.1 归约器件构造

从 3-SAT 归约到顶点覆盖：
- 给定 3-SAT 实例 φ = (x₁∨x₂∨¬x₃)∧(¬x₁∨x₂∨x₃)∧...
- 画出变量器件和子句器件
- 证明：φ 可满足 ↔ G 有大小 ≤ k 的顶点覆盖

### 1.2 近似比证明

证明以下近似算法的近似比：
- **2-近似顶点覆盖**：极大匹配的2倍覆盖是顶点覆盖
- **ln(n)-近似集合覆盖**：贪心选择的近似比 ≤ H(max|S_i|) ≤ ln(n)+1

### 1.3 归约链追踪

追踪归约链 SAT → 3-SAT → 顶点覆盖 → 独立集 → 团 → TSP：
- 每步归约保持了什么？(true-instance 映射到 true-instance)
- 归约的可计算性要求？(多项式时间)

### 1.4 采样温度

比较 SA 的温度参数 kT 与 LLM 的采样温度 T：
- 高温：接受更多劣转移 / 更均匀的 token 分布
- 低温：趋向贪心 / 趋向最高概率 token
- 它们在数学上是 Boltzmann/Gibbs 分布的不同实例

## 层次二：LLM 协同

### 2.1 归约证明审查

让 LLM 证明一个简单的归约（如 CLIQUE → 独立集）：
- LLM 的器件构造是否正确？
- 归约是否保持真值？
- 归约是否多项式时间可计算？

评价 LLM 在形式化推理中的能力边界。

### 2.2 近似算法 + LLM 启发式组合

设计一个组合策略：
- 先用近似算法得到保证质量的解（如 2-近似顶点覆盖）
- 再用 LLM 生成启发式改进（尝试删减冗余顶点）
- 验证改进后的解是否仍为顶点覆盖
- 如果是，比较解的大小

## 层次三：Agent 设计

### 3.1 归约识别器 Skill

设计归约识别器 Skill：输入问题描述，判断是否可能是 NP 完全的。

检查清单：
1. 问题是否可 naturally 表达为四核心源问题之一？（3-SAT、整数划分、顶点覆盖、哈密顿路径）
2. 是否有明显的最优化结构？
3. 是否需要全局约束而非局部贪心？

如果是 → 给出可能的归约路径和近似算法建议。
如果不是 → 可能有多项式算法，推荐搜索策略。
