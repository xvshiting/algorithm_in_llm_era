# 第9章 练习

## 层次一：基础

### 1.1 六问法实战

用六问法分析一个你不熟悉的问题（如集合覆盖、任务调度），详细记录每步推理：

1. **我真的理解问题吗？** — 定义输入/输出，构造小例子，评估规模
2. **简单算法或启发式？** — 暴力解的正确性和复杂度
3. **我的问题在目录中吗？** — 查问题分类
4. **特殊情况可解？** — 简化参数，设为平凡值
5. **哪种范式适合？** — 排序/分治/DP/贪心/随机化/NP完全性
6. **还是想不出？** — 请教专家，回到第1步

### 1.2 从简单到正确

对一个 NP 完全问题（如集合覆盖），分别给出：
- 暴力解（2^n 枚举所有子集）
- 贪心启发式
- 近似算法（ln(n)-近似）

比较三者的正确性保证和时间复杂度。

### 1.3 试飞员心态

Skiena 的比喻：即将坠毁的试飞员不恐慌——跑过清单。面试中面对设计问题：
- 不要跳跃到解决方案
- 系统地走过六问
- 记录：太多人面对问题会冻结

## 层次二：LLM 协同

### 2.1 六问法 vs LLM

让 LLM 对同一问题走六问法，你评价每步的质量：
- 第1步（理解问题）：LLM 能否准确定义输入/输出？
- 第2步（简单算法）：LLM 给出的暴力解是否正确？
- 第3步（查目录）：LLM 能否正确分类问题？
- 第4步（特殊情况）：LLM 是否能合理简化？
- 第5步（范式匹配）：LLM 的范式推荐是否合理？
- 第6步（循环）：LLM 是否会在死胡同中自我纠正？

分析：哪些步骤 LLM 表现好？哪些需要人介入？为什么？

### 2.2 简单正确 vs 复杂最优

构造一个"简单正确但慢"的 LLM 解和一个"复杂但快"的人写解：
- 暴力解：正确但 O(2^n)
- 优化解：O(n^k) 但需要仔细推导

让 LLM 自己分析两者的权衡。

## 层次三：Agent 设计

### 3.1 终章项目：六问法 Agent

把六问法编码为一个完整的 Agent Skill——这是本课程 Agent 设计的最高阶练习。

输入：自然语言问题描述
输出：推荐算法范式 + 实现框架

Agent 内部流程：
1. **问题解析**：提取输入规格、输出规格、约束条件
2. **基线生成**：生成暴力解 + 正确性自检
3. **知识库检索**：RAG 查询问题分类目录
4. **问题简化**：设为平凡值、删减参数、先简化再推广
5. **范式匹配**：分类器推荐算法范式
6. **循环重试**：如果前5步未产生满意结果，换角度重新分析

要求：
- 实现至少3个步骤的自动化
- 记录每步的推理过程
- 输出包含理由的范式推荐
