#### 4. 7.6节Ford-Fulkerson残量图初始化问题 ✅ 已修复

**原问题**：初始化残量为0，但G是邻接表，`v in G`遍历的是节点而非邻居，导致残量图初始化可能遗漏边。

**修复情况**：
```python
# 正确初始化残量图：正向边=容量，反向边=0
residual = {}
for (u, v), c in capacity.items():
    if u not in residual:
        residual[u] = {}
    if v not in residual:
        residual[v] = {}
    residual[u][v] = c  # 正向边容量
    residual[v][u] = 0  # 反向边初始为0
```

**验证结果**：
- 使用capacity字典初始化，而非邻接表 ✅
- 正向边设为容量 `c` ✅
- 反向边初始化为 `0` ✅
- 确保所有涉及节点都有残量字典 ✅

**修复状态**：完全修复

### P0-2：负权反例修复验证 ✅ 已修复

**原问题**：构造的负权反例不够经典，表述混乱。

**修复情况**：
```
    S → A → T
    S → B
    B → A

边权重：
    S→A: 1, A→T: 4, S→B: 1, B→A: -2
```

**执行过程说明**：
1. 从S开始，dist[S]=0
2. 访问S，更新邻居：dist[A]=1, dist[B]=1
3. dist[A]=dist[B]=1，两者并列最小。假设先选A访问
4. 访问A，标记为"已确认"，更新T：dist[T]=1+4=5
5. 访问B，更新A：dist[A]=1-2=-1，但A已被"已确认"，无法更新！
6. 最终结果：dist[T]=5

**实际最短路径**：S→B→A→T，距离=1-2+4=3

**验证结果**：
- 反例经典清晰 ✅
- 执行过程详细说明 ✅
- 关键洞察："已访问节点距离被负权边减小，破坏贪心选择" ✅
- 数值正确（实际最短=3，Dijkstra返回=5）✅

**修复状态**：完全修复
